4. **실행-측정 루프 필수** — 분석 결과는 반드시 실행 → 성과 재수집으로 순환

## 1. 제품 재정의

### 한 줄 정의

> X2는 **Discover → Analyze → Intent → GEO → Insight → Execute → Measure**
> 풀 루프를 하나의 플랫폼에서 완결하는 **Marketing Intelligence & Execution Platform**이다.

### 기존 v2 대비 변경

| 기존 v2 (5모듈)        | v2.1 (7스테이지 루프)              | 변경 이유                             |
| ---------------------- | ---------------------------------- | ------------------------------------- |
| 5개 독립 모듈 나열     | 7스테이지 순환 루프                | 피처링형 실행-측정 순환이 빠져 있었음 |
| Action "자동화" 까지만 | Execute + Measure 분리             | 실행과 성과 추적은 별개 도메인        |
| 단일 티어 제품         | Vertical Pack 확장 구조            | 아하트렌드형 산업별 패키징 필요       |
| 분석 결과 = 대시보드   | 근거 보고서 + 원시 데이터 내보내기 | 썸트렌드/소머즈형 UX 부재             |
| 채널 중심              | 인플루언서 탐색 + 캠페인 실행      | 피처링형 Discovery → Execute 루프     |

## 2. 7-Stage Intelligence Loop

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│  ①              ②              ③             ④                     │
│  DISCOVER ──→ ANALYZE ──→ INTENT ──→ GEO/AEO                      │
│  (탐색)       (분석)       (의도)      (AI가시성)                    │
│                                           │                         │
│                                           ▼                         │
│  ⑦              ⑥              ⑤                                   │
│  MEASURE ←── EXECUTE ←── INSIGHT                                   │
│  (측정)       (실행)       (인사이트)                                │
│     │                                                               │
│     └───────────────→ ① DISCOVER (재순환)                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 각 스테이지 정의

| #   | 스테이지     | 핵심 질문                 | 벤치마크 DNA                                            | 데이터 유형                              |
| --- | ------------ | ------------------------- | ------------------------------------------------------- | ---------------------------------------- |
| ①   | **Discover** | 누가 있고, 뭐가 뜨는가?   | 피처링(인플루언서 탐색) + 소머즈(원시 데이터 열람)      | 채널/인플루언서 DB, 트렌드 피드          |
| ②   | **Analyze**  | 수치가 어떤가? 왜 이런가? | 소머즈(원시 지표) + 썸트렌드(근거 차트)                 | ChannelSnapshot, ContentMetric, Comment  |
| ③   | **Intent**   | 시장이 뭘 원하는가?       | 리스닝마인드(인텐트 그래프, 클러스터, 저니)             | IntentQuery, KeywordResult               |
| ④   | **GEO/AEO**  | AI가 우리를 인용하는가?   | 자체(Citation 추적, 출처 우선순위)                      | AeoSnapshot, CitedSource                 |
| ⑤   | **Insight**  | 무엇을 해야 하는가?       | 썸트렌드(근거형 보고서) + 아하트렌드(Vertical 인사이트) | InsightReport, EvidenceBlock             |
| ⑥   | **Execute**  | 실제로 실행했는가?        | 피처링(캠페인 생성, 인플루언서 섭외, 콘텐츠 발행)       | Campaign, CampaignCreator, InsightAction |
| ⑦   | **Measure**  | 효과가 있었는가?          | 피처링(성과 재수집) + 소머즈(원시 성과 데이터)          | CampaignMetric, PostMeasurement          |

## 4. Vertical Pack 확장 구조 (아하트렌드/디센트릭형)

### 4.1 개념

Vertical Pack = **산업별 사전 구성된 분석 템플릿 + 키워드 세트 + 벤치마크 데이터**

기본 플랫폼은 범용이지만, Vertical Pack을 활성화하면:

- 산업별 키워드 사전 (의도 분류에 사용)
- 산업별 벤치마크 기준값 (참여율 평균, 콘텐츠 빈도 등)
- 산업별 리포트 템플릿
- 산업별 경쟁사 프리셋

### 4.2 Vertical Pack 목록

| Pack                  | 포함 내용                                                              | 타겟             |
| --------------------- | ---------------------------------------------------------------------- | ---------------- |
| **뷰티/코스메틱**     | 뷰티 키워드 500+, 성분 분석 태그, 리뷰 감성 특화, 인플루언서 티어 분류 | 뷰티 브랜드, MCN |
| **F&B**               | 음식/외식 키워드, 맛/가격/분위기 토픽 분류, 지역 기반 트렌드           | 프랜차이즈, 식품 |
| **패션/라이프스타일** | 패션 시즌 키워드, 스타일 태그, 인플루언서 스타일 매칭                  | 패션 브랜드      |
| **테크/SaaS**         | 기술 키워드, 기능 비교 인텐트, B2B 리드 감지                           | SaaS, IT         |
| **여행/호스피탈리티** | 여행지 키워드, 시즌 트렌드, 리뷰 토픽 (청결/위치/가격)                 | OTA, 호텔        |
| **금융/핀테크**       | 금융 상품 키워드, 규제 리스크 감지, 신뢰도 토픽                        | 은행, 핀테크     |
| **게임/엔터테인먼트** | 게임 타이틀 키워드, 커뮤니티 감성, 출시일 트렌드                       | 게임사, 엔터     |

### 4.3 데이터 구조

```
VerticalPack
  ├─ id, name, slug, description
  ├─ industry (enum)
  ├─ seedKeywords[]           ← 산업별 시드 키워드
  ├─ benchmarkBaseline (Json) ← 산업 평균 지표
  ├─ topicTaxonomy (Json)     ← 댓글 토픽 분류 체계
  ├─ reportTemplates[]        ← 리포트 템플릿 ID
  └─ competitorPresets[]      ← 사전 정의 경쟁사
```

## 6. 데이터 영역 분류: 실데이터 필수 vs Fallback 허용

### 절대 Mock 불가 (실데이터 필수)

| 영역                               | 이유                              |
| ---------------------------------- | --------------------------------- |
| 채널 지표 (구독자, 조회수, 참여율) | 핵심 KPI — 가짜면 제품 가치 없음  |
| 콘텐츠 목록 + 성과 지표            | 사용자가 직접 검증 가능           |
| 댓글 원문 + 감성 분석              | 원문이 없으면 분석 근거 없음      |
| 검색량/트렌드 데이터               | Google Trends/DataForSEO 실데이터 |
| AEO 인용 소스 + 순위               | Citation 추적의 핵심              |
| 캠페인 성과 지표                   | Execute → Measure 루프의 핵심     |
| 리포트 내 차트/수치                | 썸트렌드형 근거 보고서의 본질     |

### Fallback 허용 (네트워크 실패 시)

| 영역                 | Fallback 방식                  |
| -------------------- | ------------------------------ |
| AI 인사이트 텍스트   | "AI 분석 일시 불가" 상태 표시  |
| 전략 제안            | 규칙 기반 제안으로 대체        |
| 인플루언서 추천 점수 | "추천 점수 계산 중" 표시       |
| 벤치마크 비교        | "벤치마크 데이터 수집 중" 표시 |

## 7. 이후 DB에 반영해야 할 신규 엔티티 후보

### 스테이지 ① Discover 신규

```
Influencer            ← 인플루언서 프로필 (채널과 1:N)
InfluencerMetric      ← 인플루언서 지표 시계열
InfluencerTag         ← 카테고리/스타일/티어 태그
TrendFeedItem         ← 실시간 트렌드 피드 아이템
```

### 스테이지 ⑤ Insight 신규

```
EvidenceBlock         ← 리포트 내 근거 블록 (차트/표/인용)
ReportTemplate        ← 리포트 템플릿 (Vertical 포함)
ReportSection         ← 리포트 섹션 (드래그앤드롭 구성)
```

### 스테이지 ⑥ Execute 신규

```
Campaign              ← 캠페인 (목표, 기간, 예산)
CampaignCreator       ← 캠페인 × 인플루언서 매핑
CampaignContent       ← 캠페인 콘텐츠 트래킹
CreatorOutreach       ← 섭외 상태 관리
```

### 스테이지 ⑦ Measure 신규

```
CampaignMetric        ← 캠페인 성과 지표 (일별)
PostMeasurement       ← 개별 콘텐츠 성과 측정
RoiCalculation        ← ROI 계산 결과
```

### Vertical 확장

```
VerticalPack          ← 산업별 팩 정의
VerticalBenchmark     ← 산업별 벤치마크 기준값
VerticalTaxonomy      ← 산업별 토픽/태그 분류 체계
```

### 데이터 익스플로러 (소머즈형)

```
DataExportJob         ← 내보내기 작업 (CSV/Excel/API)

```

**총 신규 엔티티 후보: 17개**
