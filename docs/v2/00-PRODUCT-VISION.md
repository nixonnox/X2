## 1. 제품 정체성

**X2는 "소셜 미디어 분석 대시보드"가 아니다.**

X2는 **검색 의도(Search Intent)에서 시작해, 소셜 데이터를 연결하고, 댓글에서 시장 신호를 추출하며, AI 검색엔진(GEO/AEO)에 최적화된 콘텐츠 전략을 도출하고, 실행까지 자동화하는 Intelligence Platform**이다.

### 한 줄 정의

> "What people search → What they say → What AI answers → What you should do"

### 5대 모듈

| #   | 모듈                     | 핵심 질문                               | 기존 대비 차별점                               |
| --- | ------------------------ | --------------------------------------- | ---------------------------------------------- |
| 1   | **Search Intent**        | 사람들이 뭘 찾고 있는가?                | ListeningMind 수준 intent graph + gap analysis |
| 2   | **Social Intelligence**  | 채널/콘텐츠가 어떻게 수행되고 있는가?   | 실데이터 파이프라인 + 벤치마크                 |
| 3   | **Comment Intelligence** | 오디언스가 실제로 뭐라고 말하는가?      | 감성·토픽·리스크·FAQ 자동 추출                 |
| 4   | **GEO/AEO**              | AI 검색엔진에서 우리 콘텐츠가 나오는가? | AI Overviews/Perplexity 가시성 추적            |
| 5   | **Action Automation**    | 분석 결과로 무엇을 실행할 것인가?       | 인사이트 → 액션 아이템 → 자동 실행             |

## 2. 타겟 사용자

### Primary: 소셜 미디어 마케터 / 브랜드 매니저

- 채널 3~20개 운영
- 주간/월간 리포트 작성 필요
- 경쟁사 모니터링 수시
- "왜 이 콘텐츠가 잘됐지?" → 패턴 파악 필요

### Secondary: 에이전시 / MCN

- 클라이언트 채널 10~100개 관리
- 멀티 워크스페이스
- 화이트라벨 리포트

### Tertiary: 1인 크리에이터

- 내 채널 1~3개 최적화
- 트렌드 키워드 발굴
- 댓글 관리 + 팬 소통

## 3. 핵심 가치 제안 (Value Proposition)

| 기존 도구         | 한계                      | X2 해결               |
| ----------------- | ------------------------- | --------------------- |
| YouTube Analytics | 내 채널만, 검색 의도 없음 | 경쟁+의도+댓글 통합   |
| SocialBlade       | 표면 지표만               | 의도 기반 콘텐츠 전략 |
| ListeningMind     | 검색만, 소셜 없음         | 검색+소셜+댓글 연계   |
| BrandWatch        | 고가, 복잡                | SMB 친화적 가격/UX    |
| ChatGPT/Claude    | 실시간 데이터 없음        | 실데이터 기반 AI 분석 |

### 핵심 루프

```
[Search Intent 발견]
    ↓
[소셜 채널에서 해당 의도 대응 콘텐츠 탐색]
    ↓
[댓글에서 실제 반응/미충족 니즈 추출]
    ↓
[GEO/AEO 가시성 체크]
    ↓
[콘텐츠 전략 + 액션 아이템 자동 생성]
    ↓
[실행 후 성과 추적 → 다시 Intent 분석]
```

## 5. 기술 원칙

1. **Real Data First**: 목업/샘플 데이터 금지. 모든 화면은 실데이터 또는 "데이터 없음" 상태만 표시
2. **Pipeline-Driven**: 수집 → 정제 → 저장 → 분석 → 표시의 명확한 파이프라인
3. **Graceful Degradation**: API 키 없으면 기본 모드, AI 키 없으면 규칙 기반 분석
4. **Cost-Aware AI**: 모든 AI 호출에 비용 추적. 불필요한 호출 차단
5. **Incremental Migration**: 기존 코드를 한번에 교체하지 않고, 모듈별로 실데이터 전환
