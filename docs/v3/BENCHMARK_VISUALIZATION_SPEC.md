# Benchmark Visualization Spec

## 개요

벤치마크 baseline과 현재 측정 상태를 SVG 링/아크 형태로 시각화한다. 숫자를 직접 읽지 않아도 색상과 두께만으로 "좋음/부족/과열/주의" 상태를 직관적으로 파악할 수 있도록 설계되었다.

---

## 구조

### Center — Overall Score 원

- 전체 점수(`overallScore`)를 색상 원으로 표시
- 색상 기준:

| 점수 범위 | 색상 | 의미 |
|-----------|------|------|
| ≥ 0.7 | Emerald (`#10B981`) | 양호 |
| 0.4 ~ 0.7 | Amber (`#F59E0B`) | 보통 |
| < 0.4 | Red (`#EF4444`) | 주의 |

### Ring Segments — 메트릭별 아크 세그먼트

- 8개 메트릭별 아크 세그먼트로 구성
- 45도 간격으로 균등 배분 (360° / 8 = 45°)

### 각 세그먼트 속성

| 속성 | 결정 기준 |
|------|-----------|
| 색상 | ABOVE = Emerald, AVERAGE = Blue, BELOW = Amber |
| 두께 | `baseThickness + deviation * 15` (deviation이 클수록 두꺼움) |
| 위치 | 45도 간격 균등 배분 |

---

## 비교 모드 (Comparison Data 있을 때)

comparison data가 존재하는 경우 baseline 대비 실제값 비교 모드로 동작한다.

- Baseline 대비 실제값 비교 시각화
- Rating별 색상 구분 (ABOVE / AVERAGE / BELOW)
- 세그먼트 두께로 편차 크기 표현
- **Hover 시**: 해석 문장을 한국어로 표시
  - 예: "검색량이 업종 평균보다 23% 높습니다"
  - 예: "전환율이 기준선 대비 부족합니다"

---

## Baseline-only 모드 (Comparison Data 없을 때)

comparison data가 없는 경우 기준선만 표시하는 모드로 동작한다.

- 회색 균등 링으로 표시 (모든 세그먼트 동일 두께)
- 각 메트릭 라벨만 표시
- **안내 메시지**: "측정값 없음 — 벤치마크 기준선만 표시됩니다"
- 비교 분석 불가 상태임을 사용자에게 명확히 전달

---

## 범례

링 하단에 색상 범례를 표시한다.

| 라벨 | 색상 | 의미 |
|------|------|------|
| ABOVE | Emerald (`#10B981`) | 기준선 초과 (양호) |
| AVERAGE | Blue (`#3B82F6`) | 기준선 수준 (보통) |
| BELOW | Amber (`#F59E0B`) | 기준선 미달 (주의) |

---

## 표현 규칙

- 숫자를 몰라도 **색상**으로 상태를 직관적으로 파악할 수 있어야 한다
  - Emerald = 좋음
  - Amber = 부족 또는 주의
  - Red = 위험
- 세그먼트가 **굵으면** 차이가 크다는 것을 의미한다
  - 두꺼운 Emerald 세그먼트 = 기준 대비 크게 상회
  - 두꺼운 Amber 세그먼트 = 기준 대비 크게 하회
- 모든 세그먼트가 비슷한 두께이면 전반적으로 기준선에 근접한 상태
