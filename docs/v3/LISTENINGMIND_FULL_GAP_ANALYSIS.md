# ListeningMind vs X2 — 6대 기능 완전 비교 분석

> 작성일: 2026-03-13
> 목적: 리스닝마인드 허블의 6대 핵심 기능과 X2 현재 구현 상태를 냉정하게 비교 분석

---

## 1. 인텐트 파인더 (Intent Finder)

### 리스닝마인드 원본

| 항목        | 내용                                                                                                                                                |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 기능 정의   | 최대 100개 키워드를 입력받아 연관 키워드 + 검색 의도 + 인구통계 + 트렌드를 분석                                                                     |
| 사용자 가치 | 시장 규모 파악, 블루오션 키워드 발견, 광고비 최적화, 콘텐츠 전략 수립                                                                               |
| 입력 데이터 | 최대 100개 시드 키워드 (동시 입력)                                                                                                                  |
| 결과 데이터 | 연관 키워드별: 월간 검색량(Google+Naver), 연간 총 검색량, 성장률(%), 트렌드 스파크라인(3~4년), CPC, 광고 경쟁도, 검색 의도(I/N/C/T), 성별/연령 특성 |
| 시각화      | 시계열 트렌드 그래프(최대 4년), 토픽 자동 그룹핑, 6종 필터(토픽/의도/검색량/경쟁도/성별/연령)                                                       |
| 핵심 차별점 | 실제 Google Ads + Naver 데이터, 인구통계, 최대 100개 동시 분석                                                                                      |

### X2 현재 구현

| 항목                | 상태                                                                          |
| ------------------- | ----------------------------------------------------------------------------- |
| 키워드 입력         | 1개만 가능 (100개 동시 입력 불가)                                             |
| 검색량 데이터       | **시뮬레이션** — 수식으로 생성된 가짜 데이터 (Google Ads/Naver 미연동)        |
| 트렌드 데이터       | **시뮬레이션** — 12개월 가짜 데이터 (Google Trends/Naver DataLab 미연동)      |
| 검색 의도 분류      | 규칙 기반(정규식 매칭) 구현됨 + LLM 보조 분류 (OpenAI 연동 시)                |
| CPC/광고 경쟁도     | **없음**                                                                      |
| 인구통계(성별/연령) | **없음**                                                                      |
| 트렌드 스파크라인   | **없음** (수치만 표시)                                                        |
| 토픽 자동 그룹핑    | 클러스터링으로 대체 구현                                                      |
| 필터                | 의도 카테고리(5종) + 시간 구간(3종) 구현됨. 검색량/경쟁도/성별/연령 필터 없음 |
| 히스토리            | React 상태에만 저장 (새로고침 시 사라짐)                                      |
| 다운로드/내보내기   | **없음**                                                                      |

### 반영률: **25%**

### Gap 요약

- 실제 검색량/트렌드 데이터 없음 (가장 큰 격차)
- 복수 키워드 동시 입력 불가
- CPC, 광고 경쟁도, 인구통계 데이터 전무
- 트렌드 스파크라인 시각화 없음
- 결과 다운로드 기능 없음

## 2. 패스파인더 (Path Finder)

### 리스닝마인드 원본

| 항목        | 내용                                                                                |
| ----------- | ----------------------------------------------------------------------------------- |
| 기능 정의   | 소비자의 실제 검색 여정을 네트워크 그래프로 시각화 (최대 1,000개 검색 경로, 10단계) |
| 사용자 가치 | 경쟁 브랜드 비교 경로 파악, 구매 결정 요인 분석, 캠페인 키워드 최적화               |
| 입력 데이터 | 타겟 키워드 1개                                                                     |
| 결과 데이터 | Before/After 검색 경로, 최대 1,000개 키워드 시퀀스, 유사 키워드 자동 그룹핑         |
| 시각화      | 대형 네트워크 그래프 (before → target → after), 단계별 흐름, 클릭 인터랙션          |
| 핵심 차별점 | **실제 SERP 데이터**로 구축된 검색 경로 (추천 검색어, 관련 검색어 기반)             |

### X2 현재 구현

| 항목               | 상태                                                                      |
| ------------------ | ------------------------------------------------------------------------- |
| 네트워크 그래프    | Canvas 기반 구현됨 (3열 레이아웃: before/current/after)                   |
| 노드/링크          | 구현됨 (드래그, 줌, 클릭, 툴팁, 하이라이팅)                               |
| 검색 경로 데이터   | **시뮬레이션** — 실제 SERP/검색 경로 데이터 아님. 템플릿 기반 키워드 확장 |
| 경로 단계          | 3단계 고정 (before/current/after). 리스닝마인드는 최대 10단계             |
| 키워드 수          | 최대 150개 (리스닝마인드는 최대 1,000개)                                  |
| 유사 키워드 그룹핑 | 클러스터링으로 구현                                                       |
| 여정 경로 목록     | 구현됨 (상위 10개, weight/type 표시)                                      |
| 콘텐츠 기회 Top 5  | 구현됨                                                                    |
| 기간별 비교        | **없음** (리스닝마인드는 3~12개월 비교 가능)                              |

### 반영률: **35%**

### Gap 요약

- 핵심 격차: 실제 검색 경로 데이터 없음 (SERP 기반 수집 필요)
- 3단계 고정 → 최대 10단계로 확장 필요
- 키워드 수 150개 → 1,000개로 확장 필요
- 기간별 비교(3~12개월) 기능 없음
- 유사 키워드 자동 그룹핑 품질 개선 필요

## 3. 페르소나 뷰 (Persona View)

### 리스닝마인드 원본

| 항목        | 내용                                                                          |
| ----------- | ----------------------------------------------------------------------------- |
| 기능 정의   | 검색 행동 패턴 기반으로 소비자 페르소나를 자동 생성 (패스파인더 내 하위 기능) |
| 사용자 가치 | 데이터 기반 고객 세그먼트 발견, JTBD(해결해야 할 과업) 파악                   |
| 입력 데이터 | 패스파인더 검색 경로 데이터                                                   |
| 결과 데이터 | 동일 행동 패턴 기반 페르소나 그룹, AI 기반 인사이트                           |
| 핵심 차별점 | **검색 행동 데이터 기반** 페르소나 (인구통계가 아닌 행동 패턴)                |

### X2 현재 구현

| 항목                | 상태                                                                           |
| ------------------- | ------------------------------------------------------------------------------ |
| 페르소나 생성       | **거의 전부 하드코딩** — 의도+시간구간 조합별 5개 템플릿만 존재                |
| 레이더 차트         | 구현됨, 6축. 하지만 값의 대부분이 **하드코딩** (실제 데이터에서 파생되지 않음) |
| 핵심 질문           | **하드코딩 텍스트** — 의도 유형별 고정 문구                                    |
| 콘텐츠 전략         | **하드코딩 텍스트**                                                            |
| 파이 차트/바 차트   | 구현됨                                                                         |
| 행동 패턴 기반 분류 | **없음** — 검색 여정 데이터에서 패턴을 추출하지 않음                           |

### 반영률: **15%**

### Gap 요약

- 가장 큰 문제: 페르소나가 데이터 기반이 아님 (하드코딩 템플릿)
- 검색 행동 패턴 분석 엔진 없음
- AI/LLM 기반 페르소나 인사이트 생성 없음
- 리스닝마인드는 패스파인더의 하위 기능이지만, X2에서는 독립 페이지 (데이터 연결 없음)

## 4. 클러스터 파인더 (Cluster Finder)

### 리스닝마인드 원본

| 항목        | 내용                                                                       |
| ----------- | -------------------------------------------------------------------------- |
| 기능 정의   | 타겟 키워드 주변 최대 20,000개 관련 검색어를 SERP 기반으로 클러스터링      |
| 사용자 가치 | CEP(카테고리 진입점) 발견, TPO(시간/장소/상황) 패턴 파악, 숨겨진 수요 발굴 |
| 입력 데이터 | 타겟 키워드 1개                                                            |
| 결과 데이터 | 최대 20,000개 키워드의 의도 기반 클러스터, 클러스터별 상위 URL/도메인      |
| 시각화      | 클러스터 뷰 (네트워크 맵), 클러스터별 상세                                 |
| AI Review   | 클러스터 자동 요약, 페르소나 제안, 카피/크리에이티브 추천, 액션 브리프     |
| 핵심 차별점 | **SERP 기반 클러스터링** (검색어 유사성이 아닌 검색 결과 공유도 기반)      |

### X2 현재 구현

| 항목                 | 상태                                                         |
| -------------------- | ------------------------------------------------------------ |
| 키워드 수            | 최대 200개 (리스닝마인드는 20,000개)                         |
| 클러스터링 방식      | Louvain 알고리즘 변형 (그래프 모듈러리티 기반) — 구현됨      |
| 클러스터 목록        | 구현됨 (확장/축소, 키워드 태그, 갭 점수)                     |
| GPT 분석             | 구현됨 (OpenAI GPT-4o 연동, 폴백 포함)                       |
| 클러스터 시각화      | 바 차트만 (네트워크 맵 없음)                                 |
| 상위 URL/도메인 분석 | **없음**                                                     |
| SERP 기반 클러스터링 | **없음** — 의도+시간구간 기반 그룹핑 (SERP 결과 공유도 아님) |
| CEP/TPO 분석         | **없음**                                                     |

### 반영률: **30%**

### Gap 요약

- 핵심 격차: SERP 기반 클러스터링 없음 (현재는 의도 분류 기반)
- 키워드 규모 200개 → 20,000개로 100배 확대 필요
- 클러스터 네트워크 맵 시각화 없음
- 상위 URL/도메인 경쟁 분석 없음
- CEP/TPO 프레임 없음

## 5. 로드 뷰 (Road View)

### 리스닝마인드 원본

| 항목        | 내용                                                                  |
| ----------- | --------------------------------------------------------------------- |
| 기능 정의   | 두 키워드(시작점 → 끝점) 사이의 구체적 검색 경로를 분석               |
| 사용자 가치 | 브랜드 이탈 지점 파악, 경쟁사 전환 경로 분석, 콘텐츠 갭 식별          |
| 입력 데이터 | 시작 키워드 + 종료 키워드 (2개)                                       |
| 결과 데이터 | 두 지점 사이의 중간 검색 단계, 이탈/전환 포인트                       |
| 시각화      | 순차적 경로 시각화 (중간 단계 포함)                                   |
| 핵심 차별점 | **방향성 있는 경로 분석** — 시작점과 끝점을 지정하여 정확한 여정 추적 |

### X2 현재 구현

| 항목                  | 상태                                                                       |
| --------------------- | -------------------------------------------------------------------------- |
| 입력                  | 키워드 1개만 (시작+끝 2개 입력 불가)                                       |
| 브랜드 추출           | **단순 휴리스틱** — 시드 키워드 제거 후 첫 단어를 브랜드로 추정 (NER 없음) |
| 경쟁 분석 탭          | 구현됨 (추출된 "브랜드"별 언급 수, 갭 점수, 맥락 분류)                     |
| 콘텐츠 갭 탭          | 구현됨 (의도×시간구간 매트릭스, 블루오션 핫스팟)                           |
| 검색 여정 탭          | 구현됨 (여정 경로 목록, 시간구간 분포)                                     |
| 방향성 경로 분석      | **없음** — 두 지점 사이의 경로를 추적하지 못함                             |
| 실제 검색 경로 데이터 | **없음** — 시뮬레이션 데이터                                               |

### 반영률: **20%**

### Gap 요약

- 근본적 차이: 리스닝마인드는 2개 키워드 사이의 **방향성 경로**를 분석하지만, X2는 1개 키워드 중심 분석
- 브랜드 인식(NER) 엔진 없음 — 현재 휴리스틱은 오탐율 높음
- 실제 검색 경로 데이터 없음
- 이탈/전환 포인트 분석 없음

## 6. GPT 분석

### 리스닝마인드 원본

| 항목        | 내용                                                                       |
| ----------- | -------------------------------------------------------------------------- |
| 기능 정의   | AI가 분석 결과를 전문가 수준으로 해석하여 자연어로 설명                    |
| 사용자 가치 | 비전문가도 분석 결과 이해 가능, 즉시 활용 가능한 인사이트                  |
| 적용 범위   | 패스파인더 결과 요약, 페르소나 분석, 클러스터 라벨링, 네트워크 그래프 해석 |
| 진화        | 2025년 "ListeningMind on ChatGPT" 대화형 에이전트 출시, Claude MCP 연동    |
| 핵심 차별점 | **실제 검색 데이터 + AI 해석**의 결합                                      |

### X2 현재 구현

| 항목                 | 상태                                       |
| -------------------- | ------------------------------------------ |
| GPT 분석 API         | 구현됨 (OpenAI GPT-4o, REST + tRPC 양쪽)   |
| 클러스터 분석        | 구현됨 (요약, 상위 키워드, 페르소나, 토픽) |
| 폴백                 | 구현됨 (API 키 없을 시 규칙 기반 생성)     |
| 패스파인더 결과 분석 | **없음**                                   |
| 페르소나 AI 분석     | **없음**                                   |
| 네트워크 그래프 해석 | **없음**                                   |
| 대화형 에이전트      | **없음**                                   |
| 분석 대상            | 클러스터만 (1개 기능)                      |

### 반영률: **25%**

### Gap 요약

- 클러스터 분석만 구현 — 다른 5개 기능에 대한 AI 해석 없음
- 실제 데이터가 아닌 시뮬레이션 데이터를 분석하므로 인사이트 품질 제한
- 대화형 에이전트 기능 없음

---

## 추가 발견: 저니 파인더 (Journey Finder)

리스닝마인드에는 X2에서 "로드 뷰"로 매핑한 것과 별도로 **저니 파인더**라는 기능이 있음:

- 최대 10개 시드 키워드 입력
- 5단계 CDJ(Consumer Decision Journey) 자동 생성
- 최대 5개 경쟁 도메인 비교 (SERP 점유율)
- X2에는 **전혀 구현되지 않음**

| 기능            | 반영률   | 핵심 부족 영역                                  |
| --------------- | -------- | ----------------------------------------------- |

| 패스파인더      | 35%      | 실제 검색 경로, 10단계, 1000개 규모             |
