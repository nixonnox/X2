## S0 — Blocker (프로덕션 배포 차단)

### GAP-001: tRPC 라우터 등록 누락 — 14개 중 1개만 등록

| 항목              | 내용                                                                                                                                                                                                                                                                                                                                           |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **심각도**        | S0 (Blocker)                                                                                                                                                                                                                                                                                                                                   |
| **설계 문서**     | `DOMAIN_IMPLEMENTATION_PLAN.md`, `CORE_DATA_MODEL.md`                                                                                                                                                                                                                                                                                          |
| **실제 코드**     | `packages/api/src/root.ts`                                                                                                                                                                                                                                                                                                                     |
| **갭 설명**       | `root.ts`에 `channel` 라우터 1개만 `appRouter`에 등록되어 있음. 서비스 컨테이너에서 47개 서비스가 정상 인스턴스화되고, 14개 이상의 라우터 파일이 `packages/api/src/routers/` 디렉터리에 존재하지만, 실제로 tRPC `appRouter`에 마운트된 것은 `channel` 하나뿐임. 나머지 40/47 서비스는 API 엔드포인트가 없어 프론트엔드에서 호출 자체가 불가능. |
| **비즈니스 영향** | 대시보드, 분석, 자동화, 리포트, 온보딩, 경쟁사 분석 등 플랫폼 핵심 기능 전체가 백엔드와 단절. 사실상 channel 조회 외에는 아무 기능도 동작하지 않음.                                                                                                                                                                                            |
| **수정 복잡도**   | **Low** — 각 라우터를 `root.ts`의 `appRouter`에 추가 등록하면 됨. 라우터 파일 자체는 이미 구현되어 있으므로 import + 등록 작업만 필요.                                                                                                                                                                                                         |

### GAP-002: 프론트엔드가 Mock 데이터를 읽고 있음 — 실제 DB 데이터 미사용

| 항목              | 내용                                                                                                                                                                                                                                                                        |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **심각도**        | S0 (Blocker)                                                                                                                                                                                                                                                                |
| **설계 문서**     | `DASHBOARD_INFORMATION_ARCHITECTURE.md`, `CORE_DATA_MODEL.md`                                                                                                                                                                                                               |
| **실제 코드**     | `apps/web/src/lib/dashboard/mock-data.ts`, `apps/web/src/lib/channel/mock-data.ts` 외 6개 파일                                                                                                                                                                              |
| **갭 설명**       | 대시보드 페이지 컴포넌트들이 tRPC 쿼리 대신 `mock-data.ts`에서 하드코딩된 샘플 데이터를 직접 import하여 렌더링. 백엔드 수집 파이프라인(Collection Pipeline)은 실제로 소셜 미디어 API를 호출하여 DB에 데이터를 적재하고 있으나, 프론트엔드가 이 데이터를 전혀 참조하지 않음. |
| **비즈니스 영향** | 사용자가 보는 모든 대시보드 수치, 차트, 분석 결과가 가짜 데이터. 실제 수집된 데이터가 DB에 존재하더라도 화면에 반영되지 않으므로, 플랫폼의 핵심 가치 제안(실시간 소셜 분석)이 완전히 무효화됨.                                                                              |
| **수정 복잡도**   | **Medium** — GAP-001 해소 후, 각 페이지 컴포넌트에서 mock import를 tRPC `useQuery` 호출로 교체해야 함. 6개 이상의 페이지/컴포넌트 수정 필요.                                                                                                                                |

## 종합 평가

### 핵심 진단

플랫폼의 **백엔드 서비스 레이어**(분석 엔진, 서비스 컨테이너, 수집 파이프라인)는 높은 완성도를 보이나, **프론트엔드-백엔드 연결 계층**이 사실상 끊어져 있는 상태. 구체적으로:

1. **tRPC 라우터 미등록** (GAP-001)으로 인해 백엔드 서비스에 접근할 API 경로 자체가 없음
2. **Mock 데이터 의존** (GAP-002, GAP-005)으로 인해 프론트엔드가 백엔드를 호출할 필요 자체를 느끼지 않음
3. 결과적으로 **"엔진은 돌아가지만 아무도 그 결과를 보지 못하는"** 상태
