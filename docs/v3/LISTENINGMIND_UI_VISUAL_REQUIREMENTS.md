# ListeningMind UI/UX 및 시각화 요구사항

> 작성일: 2026-03-13
> 목적: 6대 기능별 UX 요구사항과 시각화 요구사항 정리

---

## 1. 인텐트 파인더 화면

### 현재 구현된 것

- [O] 단일 키워드 입력
- [O] 의도 분류별 도넛 차트
- [O] 시간구간별 도넛 차트
- [O] 콘텐츠 갭 매트릭스 바 차트
- [O] 블루오션 키워드 목록
- [O] 필터 (의도 카테고리 5종 + 시간구간 3종)
- [O] 키워드 분류 테이블 (키워드, 검색량, 트렌드, 의도, 구간, 갭점수, 소셜볼륨)

### 추가 필요한 UX

| 요구사항                | 설명                                                                         | 우선순위 |
| ----------------------- | ---------------------------------------------------------------------------- | -------- |
| **복수 키워드 입력**    | 태그 입력 방식으로 최대 100개 키워드 동시 입력. 쉼표/엔터로 구분             | 높음     |
| **트렌드 스파크라인**   | 각 키워드 행에 미니 라인 차트 (12~48개월). 빨간/파란 색상으로 상승/하락 표시 | 높음     |
| **검색량 범위 필터**    | 슬라이더로 최소~최대 검색량 범위 설정                                        | 중간     |
| **광고 경쟁도 필터**    | 낮음/중간/높음 3단계 필터                                                    | 중간     |
| **검색 의도 분류 개선** | 현재 5분류(discovery 등) → 4분류(I/N/C/T)로 통일 또는 병행                   | 중간     |
| **정렬 기능**           | 테이블 헤더 클릭으로 검색량/갭점수/CPC 등 기준 정렬                          | 중간     |
| **페이지네이션**        | 현재 50개 제한 → 전체 데이터 페이지네이션 또는 가상 스크롤                   | 중간     |
| **CSV/Excel 다운로드**  | 키워드 테이블 전체 데이터 내보내기                                           | 중간     |
| **히스토리 영속화**     | 현재 메모리만 → DB 저장 + 최근 분석 목록 유지                                | 중간     |
| **토픽 자동 그룹핑 뷰** | 키워드를 토픽(클러스터)별로 그룹화하여 보여주는 토글 뷰                      | 낮음     |
| **키워드 연결 액션**    | 각 키워드에서 패스파인더/클러스터파인더로 바로 이동 버튼                     | 낮음     |

### 시각화 요구사항

| 차트                  | 현재          | 목표                                    |
| --------------------- | ------------- | --------------------------------------- |
| 의도 분류             | 도넛 PieChart | 유지                                    |
| 시간구간 분포         | 도넛 PieChart | 유지                                    |
| 갭 매트릭스           | 수평 BarChart | 유지                                    |
| **트렌드 스파크라인** | 없음          | 테이블 내 미니 LineChart (각 키워드 행) |
| **시계열 트렌드**     | 없음          | 선택한 키워드 5개의 12~48개월 라인 차트 |
| **토픽 트리맵**       | 없음 (삭제됨) | 토픽별 키워드 규모 시각화 (Treemap)     |

## 2. 패스파인더 네트워크 그래프

### 현재 구현된 것

- [O] Canvas 기반 네트워크 그래프 (3열: before/current/after)
- [O] 드래그, 줌, 클릭, 툴팁
- [O] 노드 색상 = 의도 카테고리, 크기 = 검색량
- [O] 선택한 노드의 연결 키워드 상세
- [O] 여정 단계 분포 바 차트
- [O] 시간구간 분석 바 차트
- [O] 필터 (시간구간 + 의도)

### 추가 필요한 UX

| 요구사항                 | 설명                                                             | 우선순위 |
| ------------------------ | ---------------------------------------------------------------- | -------- |
| **다단계 경로 표시**     | 현재 3단계 고정 → 최대 10단계로 확장 가능한 수평 스크롤 레이아웃 | 높음     |
| **키워드 규모 확대**     | 최대 150개 → 최대 1,000개 노드 렌더링 성능                       | 높음     |
| **유사 키워드 그룹핑**   | 비슷한 검색어를 하나의 "슈퍼노드"로 합쳐서 표시 (축소/확장 가능) | 높음     |
| **로드뷰 연결 버튼**     | 노드 2개를 선택하면 "이 경로 보기(로드뷰)" 버튼 표시             | 중간     |
| **기간 비교**            | 3개월/6개월/12개월 전 데이터와 현재 비교하는 타임라인 슬라이더   | 중간     |
| **노드 검색**            | 그래프 내 특정 키워드 검색 + 하이라이팅                          | 중간     |
| **미니맵**               | 전체 그래프의 축소 미리보기 (현재 뷰포트 위치 표시)              | 낮음     |
| **스크린샷/이미지 저장** | 현재 그래프를 PNG로 다운로드                                     | 낮음     |

### 시각화 요구사항

| 항목          | 현재                 | 목표                                             |
| ------------- | -------------------- | ------------------------------------------------ |
| 그래프 렌더러 | 커스텀 Canvas (2D)   | 유지하되, 1000+ 노드 성능 최적화 (쿼드트리, LOD) |
| 레이아웃      | 3열 고정             | 다단계 수평 흐름 (스크롤 가능)                   |
| 엣지 표현     | 베지어 곡선 + 화살표 | 유지                                             |
| 노드 크기     | log(검색량)          | 유지                                             |
| 인터랙션      | 드래그/줌/클릭       | 유지 + 멀티셀렉트 추가                           |

## 3. 페르소나 뷰

### 현재 구현된 것

- [O] 페르소나 분포 파이 차트
- [O] 의도 분포 바 차트
- [O] 페르소나 카드 그리드 (레이블, 설명, 비율, 의도/구간 배지)
- [O] 레이더 차트 (6축)
- [O] 핵심 질문, 콘텐츠 전략, 관련 키워드

### 추가 필요한 UX

| 요구사항                 | 설명                                                                | 우선순위 |
| ------------------------ | ------------------------------------------------------------------- | -------- |
| **데이터 기반 페르소나** | 현재 하드코딩 → 실제 검색 행동 패턴에서 추출                        | 높음     |
| **AI 생성 인사이트**     | 페르소나별 GPT 분석 (왜 이 검색을 하는지, 어떤 콘텐츠가 효과적인지) | 높음     |
| **검색 여정 시각화**     | 각 페르소나의 대표 검색 경로를 미니 플로우 차트로 표시              | 중간     |
| **페르소나 비교**        | 2개 페르소나를 나란히 비교하는 뷰                                   | 낮음     |
| **레이더 차트 데이터**   | 현재 하드코딩 값 → 실제 행동 데이터에서 산출                        | 높음     |

### 시각화 요구사항

| 차트                 | 현재           | 목표                                       |
| -------------------- | -------------- | ------------------------------------------ |
| 페르소나 분포        | 도넛 PieChart  | 유지                                       |
| 레이더 차트          | 6축 RadarChart | 유지, 데이터 소스 개선                     |
| **행동 여정 미니맵** | 없음           | 페르소나별 대표 검색 경로 미니 플로우 차트 |
| **JTBD 매핑**        | 없음           | 페르소나별 해결하려는 과업(JTBD) 카드      |

## 4. 클러스터 파인더 뷰

### 현재 구현된 것

- [O] 클러스터 크기 바 차트
- [O] 클러스터 갭 점수 바 차트
- [O] 클러스터 목록 (확장/축소, 키워드 태그)
- [O] GPT 종합 분석 (요약, 상위 키워드, 페르소나, 토픽)

### 추가 필요한 UX

| 요구사항                 | 설명                                                                | 우선순위 |
| ------------------------ | ------------------------------------------------------------------- | -------- |
| **클러스터 네트워크 맵** | 클러스터 간 관계를 보여주는 네트워크 시각화 (버블 또는 포스 그래프) | 높음     |
| **키워드 규모 확대**     | 200개 → 최대 20,000개 키워드 처리                                   | 높음     |
| **상위 도메인 분석**     | 클러스터별 SERP 상위 도메인/URL 표시                                | 중간     |
| **CEP/TPO 라벨링**       | 각 클러스터에 카테고리 진입점 / 시간·장소·상황 라벨                 | 중간     |
| **클러스터 비교**        | 2개 클러스터를 나란히 비교                                          | 낮음     |
| **Treemap 시각화**       | 클러스터 크기를 한눈에 보는 트리맵 (기존에 삭제된 기능 복원)        | 낮음     |

### 시각화 요구사항

| 차트                  | 현재     | 목표                                                           |
| --------------------- | -------- | -------------------------------------------------------------- |
| 클러스터 크기         | BarChart | 유지                                                           |
| 갭 점수               | BarChart | 유지                                                           |
| **클러스터 네트워크** | 없음     | 포스 그래프 또는 버블 차트 (클러스터 = 버블, 크기 = 키워드 수) |
| **도메인 점유율**     | 없음     | 클러스터별 수평 BarChart (도메인 x 노출 비율)                  |

## 5. 로드 뷰

### 현재 구현된 것

- [O] 3탭 구조 (검색 여정 / 경쟁 분석 / 콘텐츠 갭)
- [O] 시간구간 분포 바 차트
- [O] 검색 맥락 분포 파이 차트
- [O] 여정 경로 목록
- [O] 경쟁사 확장 목록 (멘션 수, 갭 점수)
- [O] 콘텐츠 갭 매트릭스 테이블
- [O] 블루오션 핫스팟 카드

### 추가 필요한 UX

| 요구사항               | 설명                                                   | 우선순위 |
| ---------------------- | ------------------------------------------------------ | -------- |
| **2-키워드 입력**      | 시작 키워드 + 끝 키워드 2개 입력 (로드 뷰 핵심)        | 높음     |
| **방향성 경로 시각화** | A→B 사이 중간 단계를 순차적 플로우로 표시              | 높음     |
| **이탈 포인트 표시**   | 경로에서 다른 방향으로 빠지는 분기점을 시각적으로 강조 | 중간     |
| **브랜드 인식 개선**   | 현재 단순 휴리스틱 → 브랜드 사전 또는 NER 기반         | 중간     |
| **경쟁 도메인 비교**   | 최대 5개 도메인의 CDJ 단계별 SERP 점유율 비교          | 낮음     |

### 시각화 요구사항

| 차트               | 현재     | 목표                                          |
| ------------------ | -------- | --------------------------------------------- |
| 시간구간 분포      | BarChart | 유지                                          |
| 맥락 분포          | PieChart | 유지                                          |
| **방향성 경로**    | 없음     | 수평 스텝 플로우 차트 (A → 중간1 → 중간2 → B) |
| **이탈 분기 표시** | 없음     | 플로우 차트에서 분기 화살표로 표시            |
| **도메인 점유율**  | 없음     | CDJ 단계별 stacked BarChart                   |

## 6. GPT 분석 결과 화면

### 현재 구현된 것

- [O] 클러스터 종합 분석 텍스트
- [O] 상위 키워드 태그
- [O] 페르소나 카드 (레이블, 상황, 핵심 질문)
- [O] 토픽 분석 목록 (질문 + 근거)
- [O] 로딩/에러 상태 표시

### 추가 필요한 UX

| 요구사항               | 설명                                                                   | 우선순위 |
| ---------------------- | ---------------------------------------------------------------------- | -------- |
| **전체 기능 GPT 분석** | 클러스터뿐 아니라 패스파인더, 페르소나, 로드 뷰 결과에도 GPT 분석 버튼 | 높음     |
| **분석 결과 저장**     | GPT 분석 결과를 DB에 저장하여 재열람 가능                              | 중간     |
| **마크다운 포맷**      | GPT 응답을 마크다운으로 렌더링 (볼드, 리스트, 헤딩 등)                 | 중간     |
| **질문 추천**          | "이 데이터에 대해 더 물어볼 수 있는 질문" 제안                         | 낮음     |
| **대화형 Q&A**         | 분석 결과에 대해 후속 질문하는 채팅 인터페이스                         | 낮음     |

### 시각화 요구사항

| 항목              | 현재               | 목표                                   |
| ----------------- | ------------------ | -------------------------------------- |
| 분석 결과         | 텍스트 카드 + 태그 | 유지 + 마크다운 렌더링 추가            |
| 페르소나 카드     | 3열 그리드         | 유지                                   |
| **인사이트 요약** | 없음               | 핵심 인사이트 3개를 시각적 카드로 강조 |

## 공통 UX 요구사항

| 항목          | 현재                        | 목표                               |
| ------------- | --------------------------- | ---------------------------------- |
| 결과 내보내기 | 없음                        | CSV/Excel 다운로드 (전체 페이지)   |
| 결과 영속화   | 메모리만 (새로고침 시 소실) | DB 저장 + 분석 히스토리 관리       |
| 기능 간 연결  | 독립적 페이지               | 키워드 클릭 시 다른 기능으로 이동  |
| 로딩 UX       | Loader2 스피너              | 단계별 프로그레스 바 (12단계 표시) |
| 반응형        | 미확인                      | 태블릿/모바일 대응                 |
| 다크 모드     | CSS 변수 기반 (부분 적용)   | 전체 적용 확인                     |
