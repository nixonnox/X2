## 2. Pipeline Monitor (수집 상태 관제)

### 2.1 대시보드 지표

| 지표         | 데이터 소스                        | 표시     | 임계값                 |
| ------------ | ---------------------------------- | -------- | ---------------------- |
| 활성 채널 수 | `Channel WHERE status=ACTIVE`      | 숫자     | —                      |
| 수집 성공률  | `ScheduledJob 최근 24시간`         | 백분율   | < 90% 경고, < 70% 위험 |
| 마지막 수집  | `Channel.lastSyncedAt`             | 시간 ago | > 24시간 경고          |
| 실패 Job 수  | `ScheduledJob WHERE status=FAILED` | 숫자     | > 0 경고               |
| 큐 대기 수   | BullMQ waiting count               | 숫자     | > 100 경고             |
| API 할당량   | 플랫폼별 API 사용량                | 잔여 %   | < 20% 경고             |

### 2.2 채널별 수집 상태

```
┌────────────────────────────────────────────────────────────┐
│ Channel Sync Status                                        │
│                                                            │
│ Channel          Platform   Status    Last Sync    Errors  │
│ ─────────────────────────────────────────────────────────  │
│ @brand_kr        YouTube    ✅ OK     2분 전        0      │
│ @brand_insta     Instagram  ✅ OK     3시간 전      0      │
│ @competitor_a    YouTube    ⚠️ SLOW   25시간 전     0      │
│ @brand_tiktok    TikTok     ❌ FAIL   3일 전        3      │
│                                                            │
│ [필터: platform, status, lastSync]                         │
└────────────────────────────────────────────────────────────┘
```

### 2.3 Job 실행 이력

| 필드       | 설명                                          |
| ---------- | --------------------------------------------- |
| jobId      | ScheduledJob ID                               |
| type       | CHANNEL_SYNC, CONTENT_SYNC, COMMENT_SYNC, ... |
| status     | ACTIVE, PAUSED, FAILED, COMPLETED             |
| lastRunAt  | 마지막 실행 시간                              |
| lastError  | 에러 메시지 (있는 경우)                       |
| retryCount | 재시도 횟수                                   |
| duration   | 실행 시간 (ms)                                |

## 3. AI Monitor (AI 상태 관제)

### 3.1 대시보드 지표

| 지표                | 데이터 소스                      | 표시             | 임계값              |
| ------------------- | -------------------------------- | ---------------- | ------------------- |
| 일별 AI 토큰 사용량 | `UsageMetric.aiTokensUsed`       | 숫자 + 추이 차트 | 플랜 한도 80% 경고  |

## 4. Report Monitor (리포트/이메일/공유 관제)

### 4.1 대시보드 지표

| 지표             | 데이터 소스                  | 표시        |
| ---------------- | ---------------------------- | ----------- |
| 생성된 리포트 수 | `InsightReport COUNT`        | 일별/주별   |
| 공유 링크 접속   | shareToken 접속 로그         | 일별        |
| 이메일 발송 수   | 이메일 서비스 로그           | 일별 (가설) |
| 이메일 발송 실패 | 이메일 서비스 에러           | 숫자        |
| 데이터 내보내기  | `DataExportJob` 상태별       | 숫자        |
| 리포트 상태 분포 | DRAFT / PUBLISHED / ARCHIVED | 파이 차트   |

### 4.2 데이터 내보내기 상태

```
DataExportJob Status:
  QUEUED     → 대기 중 (예상 시간 표시)
  PROCESSING → 처리 중 (진행률 표시)
  READY      → 완료 (다운로드 링크, 만료 시간)
  EXPIRED    → 만료 (재요청 유도)
  FAILED     → 실패 (에러 메시지, 재시도 버튼)
```

## 6. 실패/재시도/알림 구조

### 6.1 재시도 정책

```
Job 실패 시:
  1회차: 즉시 재시도
  2회차: 1분 후 재시도
  3회차: 5분 후 재시도
  4회차 이상: Job 일시정지 (PAUSED)
            → 관리자 알림
            → 수동 재시작 또는 자동 복구 (다음 스케줄)

Auth 실패 시:
  즉시 Channel.status = ERROR
  사용자에게 재인증 요청 알림
  재인증 완료 시 자동 재수집

Rate Limit 시:
  지수 백오프 (1분, 2분, 4분, 8분, 최대 30분)
  할당량 리셋 후 자동 재개
```

### 6.2 알림 채널

| 이벤트               | 대상            | 채널                | 우선순위 |
| -------------------- | --------------- | ------------------- | -------- |
| 수집 3회 연속 실패   | Workspace Admin | 인앱 + 이메일       | HIGH     |
| AI 토큰 80% 도달     | Workspace Admin | 인앱                | MEDIUM   |
| API 할당량 90% 도달  | System Admin    | 인앱 + Slack (내부) | HIGH     |
| OAuth 토큰 만료      | Channel 소유자  | 인앱 + 이메일       | HIGH     |
| 리포트 생성 완료     | 요청자          | 인앱                | LOW      |
| 캠페인 성과 업데이트 | 캠페인 담당자   | 인앱                | LOW      |
| 부정 감성 급증       | Workspace Admin | 인앱 + 이메일       | HIGH     |
| 플랜 한도 초과       | Workspace Owner | 인앱 + 이메일       | MEDIUM   |
