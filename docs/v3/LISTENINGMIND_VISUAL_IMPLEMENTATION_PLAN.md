## 1. 패스파인더 네트워크 그래프

### 현재 구현 상태

| 항목 | 상태 | 상세 |
|------|------|------|
| 렌더링 엔진 | Custom Canvas 2D | HTML `<canvas>` 직접 드로잉 |
| 레이아웃 | 3열 고정 (18%/50%/82%) | before/current/after 열 |
| 노드 | 원형, log(volume) 크기 | 의도별 색상, rising 표시(초록 점) |
| 엣지 | 베지어 곡선 | hover 시 화살표 표시 |
| 인터랙션 | 드래그, 줌(0.3x~3x), 클릭 | 선택 노드 하이라이팅 |
| 노드 수 | 최대 150개 | 열당 20개 제한 |
| 라벨 표시 | 조건부 (시드/선택/volume>500) | — |
| 보조 차트 | 여정 단계 분포 BarChart, 시간구간 PieChart | Recharts 사용 |
| 콘텐츠 기회 | Top 5 리스트 | opportunity score 정렬 |

### 신규 구현 범위

| 항목 | 목표 | 구현 방식 | 우선순위 |
|------|------|-----------|---------|
| **다단계 레이아웃** | 3열 → 최대 10열 수평 스크롤 | Canvas 뷰포트 확장 + 횡스크롤 | 높음 |
| **1,000 노드 성능** | 150 → 1,000개 | 쿼드트리 인덱싱 + 뷰포트 컬링 + LOD | 높음 |
| **슈퍼노드** | 유사 키워드 그룹을 하나의 노드로 | 클릭 시 펼침/접힘, 그룹 내 노드 표시 | 높음 |
| **멀티셀렉트** | 노드 2개 선택 → 로드뷰 연결 | Shift+클릭, "경로 보기" 버튼 | 중간 |
| **노드 검색** | 그래프 내 키워드 검색 + 하이라이팅 | 검색바 + 자동완성 + 화면 이동 | 중간 |
| **기간 비교** | 3/6/12개월 전 데이터 비교 | 타임라인 슬라이더, 변화 노드 강조 | 중간 |
| **미니맵** | 전체 그래프 축소 미리보기 | Canvas inset (우하단) | 낮음 |
| **PNG 내보내기** | 그래프 이미지 저장 | `canvas.toDataURL()` | 낮음 |

### Canvas 아키텍처 개선

```
현재:
  drawNodes() → 모든 노드를 매 프레임 그림 (O(n))
  findNodeAt(x,y) → 모든 노드 순회 (O(n))

목표:
  QuadTree:
    insert(node) / query(rect) → O(log n) 공간 쿼리
    마우스 이벤트 시 뷰포트 영역만 검색

  LOD (Level of Detail):
    줌 > 1.5x: 모든 라벨 + 세부 정보
    줌 0.8~1.5x: 주요 노드 라벨만
    줌 < 0.8x: 노드 점만 (라벨 없음)

  뷰포트 컬링:
    화면 밖 노드는 draw() 호출 스킵
    엣지는 양쪽 노드 중 하나라도 화면 안이면 그림

  다단계 레이아웃:
    열 수 = min(maxSteps * 2 + 1, 10)
    열 간격 = canvasWidth / 열 수
    스크롤: 전체 너비 > 뷰포트일 때 횡스크롤 활성화
```

## 2. 로드뷰 방향성 흐름도

### 현재 구현 상태

| 항목 | 상태 | 상세 |
|------|------|------|
| 입력 | 키워드 1개 | 시작+끝 2개 불가 |
| 탭 구조 | 3탭 (여정/경쟁/갭) | 구현됨 |
| 경로 시각화 | **없음** | 목록만 표시 (플로우 차트 없음) |
| 시간구간 분포 | BarChart | Recharts |
| 검색 맥락 | PieChart (direct/comparison/alternative/related) | Recharts |
| 경쟁사 분석 | 확장 리스트 (브랜드별 멘션, 갭점수) | — |
| 콘텐츠 갭 | 매트릭스 테이블 + 블루오션 핫스팟 | — |
| 브랜드 추출 | 단순 휴리스틱 | 시드 키워드 제거 → 첫 단어 |

### 신규 구현 범위

| 항목 | 목표 | 구현 방식 | 우선순위 |
|------|------|-----------|---------|
| **2-키워드 입력** | 시작 + 끝 키워드 입력 | 입력 필드 2개 + "경로 분석" 버튼 | 높음 |
| **방향성 경로 플로우** | A → 중간1 → 중간2 → B 시각화 | 수평 스텝 플로우 차트 (SVG) | 높음 |
| **이탈 포인트 표시** | 분기 화살표로 이탈 경로 표시 | 플로우 차트 내 분기 노드 강조 | 중간 |
| **대안 경로** | 최대 5개 대안 경로 표시 | 탭 또는 토글로 경로 전환 | 중간 |
| **브랜드 NER** | 키워드에서 브랜드 자동 인식 | 브랜드 사전 + 패턴 매칭 | 중간 |
| **도메인 점유율** | CDJ 단계별 도메인 SERP 점유율 | Stacked BarChart | 낮음 |

### 방향성 경로 플로우 차트 설계

```
시각화 구조:

  [시작: 화장품]  →  [중간1: 화장품 추천]  →  [중간2: 화장품 브랜드 비교]  →  [끝: 올리브영]
       │                    │                         │
       │              ↗ [이탈: 스킨케어 루틴]     ↗ [이탈: 화장품 성분 분석]
       │
       └─ 대안 경로: [화장품] → [화장품 순위] → [올리브영 추천]  → [올리브영]

구현 방식:
  - SVG 기반 (Canvas 대신 — DOM 인터랙션이 더 중요)
  - 수평 스텝 레이아웃
  - 주 경로: 굵은 선 + 강조 색상
  - 이탈 경로: 점선 + 회색
  - 대안 경로: 가는 실선 + 연한 색상
  - 각 노드: 키워드 + 검색량 + 의도 배지
  - 엣지: 화살표 + 전환 강도 (선 굵기)
  - 분기 노드: 다이아몬드 모양 또는 빨간 테두리
```

## 3. 페르소나 뷰 그룹 시각화

### 현재 구현 상태

| 항목 | 상태 | 상세 |
|------|------|------|
| 페르소나 분포 | PieChart (percentage 기준) | Recharts |
| 의도 분포 | BarChart (전체 키워드 의도) | Recharts |
| 페르소나 카드 | 그리드 레이아웃 | 레이블, 설명, 비율, 의도/구간 배지 |
| 레이더 차트 | 6축 RadarChart | **값이 하드코딩** |
| 핵심 질문 | 불릿 리스트 | **텍스트 하드코딩** |
| 콘텐츠 전략 | 텍스트 단락 | **텍스트 하드코딩** |
| 관련 키워드 | 태그 목록 (최대 15개) | 클러스터 키워드에서 추출 |

### 신규 구현 범위

| 항목 | 목표 | 구현 방식 | 우선순위 |
|------|------|-----------|---------|
| **데이터 기반 레이더** | 하드코딩 → 실제 비율 산출 | `calculateTraits()` 함수 | 높음 |
| **AI 인사이트 카드** | GPT 생성 JTBD, 고충점, 동기 | LLM 레이블러 결과 표시 | 높음 |
| **행동 여정 미니맵** | 페르소나별 대표 검색 경로 | 미니 플로우 차트 (SVG, 3~5노드) | 중간 |
| **JTBD 매핑 카드** | 해결하려는 과업 시각화 | 카드 그리드 (아이콘 + 텍스트) | 중간 |
| **페르소나 비교** | 2개 나란히 비교 | 좌우 분할 레이아웃 + 겹침 레이더 | 낮음 |

### 행동 여정 미니맵 설계

```
각 페르소나 카드 내부에 표시:

  [정보 탐색] ─→ [비교] ─→ [가격 확인] ─→ [구매]
    ● ──────── ● ──────── ● ──────── ●

  - 3~5개 노드의 미니 플로우
  - 각 노드 = 의도 카테고리 아이콘
  - 노드 크기 = 해당 단계의 키워드 비율
  - SVG 인라인 (80px 높이)
  - 호버 시 키워드 목록 팝오버
```

### 신규 구현 범위

| 항목 | 목표 | 구현 방식 | 우선순위 |
|------|------|-----------|---------|
| **클러스터 네트워크 맵** | 클러스터 간 관계 시각화 | 포스 그래프 또는 버블 차트 | 높음 |
| **도메인 점유율** | 클러스터별 SERP 상위 도메인 | 수평 BarChart (도메인별) | 중간 |
| **CEP/TPO 라벨** | 각 클러스터에 진입점/상황 태그 | 배지 UI (자동 라벨링) | 중간 |
| **Treemap** | 클러스터 크기 한눈에 보기 | Recharts Treemap | 낮음 |
| **클러스터 비교** | 2개 클러스터 나란히 비교 | 모달 또는 분할 뷰 | 낮음 |

### 클러스터 네트워크 맵 설계

```
2가지 시각화 옵션:

[옵션 A] 버블 차트 (Bubble Chart)
  - 각 버블 = 클러스터
  - 버블 크기 = 키워드 수
  - 버블 색상 = dominantIntent
  - 버블 위치 = SERP 유사도 기반 (MDS 차원 축소)
  - 클릭 시 클러스터 상세 패널 열림
  - 장점: 직관적, 렌더링 빠름
  - 단점: 클러스터 간 관계 표현 약함

[옵션 B] 포스 그래프 (Force-Directed Graph)
  - 노드 = 클러스터 (크기 = 키워드 수)
  - 엣지 = 클러스터 간 SERP 유사도 (임계값 이상만)
  - 엣지 굵기 = 유사도 강도
  - 포스 시뮬레이션: d3-force 또는 Canvas 기반
  - 장점: 관계 명확, 인터랙티브
  - 단점: 렌더링 복잡

추천: 옵션 B (포스 그래프)
  - 리스닝마인드와 동일한 형태
  - d3-force 활용 (기존 Canvas 엔진과 통합 가능)
  - 노드 수가 클러스터 수 (20~50개) → 성능 문제 없음
```

**포스 그래프 구현 스펙:**

```typescript
interface ClusterNetworkNode {
  id: string;                          // cluster ID
  label: string;                       // cluster name
  size: number;                        // keyword count
  color: string;                       // intent color
  avgGapScore: number;
  dominantIntent: IntentCategory;
  x?: number;                          // force simulation position
  y?: number;
}

interface ClusterNetworkEdge {
  source: string;                      // cluster ID
  target: string;                      // cluster ID
  serpSimilarity: number;              // 0-1
  sharedDomains: string[];             // 공유 도메인
}

// 렌더링:
// - Canvas 또는 SVG (클러스터 수 < 50이면 SVG가 인터랙션에 유리)
// - d3-force: forceCenter, forceManyBody, forceLink(serpSimilarity)
// - 노드 클릭 → 사이드 패널에 클러스터 상세 + GPT 분석
// - 엣지 호버 → 공유 도메인 툴팁
```

## 5. 공통 시각화 요구사항

### 5.1 트렌드 스파크라인 (인텐트 파인더 테이블)

```
현재: 없음 (숫자만 표시)
목표: 각 키워드 행에 미니 라인 차트

구현:
  - Recharts ResponsiveContainer (너비 80px, 높이 30px)
  - 12~48개월 데이터 포인트
  - 상승: 파란색 선, 하락: 빨간색 선
  - 축/라벨 없음 (미니 차트)
  - 호버 시 툴팁 (날짜 + 검색량)

대안: SVG inline으로 직접 그리기 (라이브러리 없이)
  - 테이블에 50개 행 × 각 1개 차트 = 50개 차트
  - Recharts보다 SVG inline이 렌더링 성능 유리
```

### 5.2 CSV/Excel 다운로드

```
적용 범위: 인텐트 파인더, 클러스터 파인더, 로드뷰

구현:
  - 헤더 버튼 "내보내기" → CSV 다운로드
  - 라이브러리: 없음 (Blob + URL.createObjectURL)
```
