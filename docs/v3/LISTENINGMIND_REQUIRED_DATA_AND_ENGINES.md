# ListeningMind 구현을 위한 데이터/엔진/저장 구조 요구사항

> 작성일: 2026-03-13
> 목적: 각 기능 구현에 필요한 데이터 소스, 분석 엔진, 저장 구조를 정의

---

## 1. 필요한 외부 데이터 소스

### A. 필수 (모든 기능의 기반)

| 데이터 소스                        | 제공 데이터                           | 사용 기능                      | 비용/제약                                        |
| ---------------------------------- | ------------------------------------- | ------------------------------ | ------------------------------------------------ |
| **Google Ads Keyword Planner API** | 월간 검색량, CPC, 광고 경쟁도, 시즌성 | 인텐트 파인더, 클러스터 파인더 | Google Ads 계정 필요, 광고 지출 시 정확한 데이터 |
| **Google Autocomplete API**        | 자동완성 추천 검색어                  | 패스파인더, 클러스터 파인더    | 비공식 API, 속도 제한 주의                       |
| **Google Related Searches**        | "관련 검색어", "다른 사용자 검색"     | 패스파인더, 클러스터 파인더    | SERP 크롤링 또는 SerpAPI 필요                    |
| **Naver Search API**               | 검색량, 자동완성, 연관 검색어         | 인텐트 파인더                  | 25,000 calls/day                                 |

### B. 권장 (품질 향상)

| 데이터 소스              | 제공 데이터                              | 사용 기능                                           | 비용/제약                      |
| ------------------------ | ---------------------------------------- | --------------------------------------------------- | ------------------------------ |
| **SerpAPI / DataForSEO** | SERP 결과, PAA, 스니펫, 도메인 순위      | 클러스터 파인더 (SERP 기반 클러스터링), 도메인 분석 | 유료 ($50~200/월)              |
| **Google Trends API**    | 트렌드 스파크라인, 시즌성, 지역별 관심도 | 인텐트 파인더                                       | pytrends (비공식) 또는 SerpAPI |
| **Naver DataLab API**    | 네이버 트렌드, 성별/연령 관심도          | 인텐트 파인더 (인구통계)                            | 공식 API                       |

### C. 선택 (부가 가치)

| 데이터 소스         | 제공 데이터               | 사용 기능 |
| ------------------- | ------------------------- | --------- |
| YouTube Data API    | 콘텐츠 수, 조회수, 경쟁도 | 갭 분석   |
| Instagram Graph API | 해시태그 볼륨             | 갭 분석   |
| TikTok Research API | 해시태그 볼륨, 트렌드     | 갭 분석   |

## 2. 필요한 분석 엔진

### 엔진 1: 키워드 확장 엔진 (Keyword Expansion Engine)

| 항목        | 내용                                                              |
| ----------- | ----------------------------------------------------------------- |
| 목적        | 시드 키워드에서 연관 키워드를 실제로 수집                         |
| 현재 상태   | `pipeline/keyword-expander.ts` — 전체 시뮬레이션 (템플릿 기반)    |
| 필요한 변경 | Google Autocomplete + Related Searches + Naver 자동완성 실제 연동 |
| 입력        | 시드 키워드 1~100개                                               |
| 출력        | 확장된 키워드 목록 (최대 20,000개) + 출처(source) + 깊이(depth)   |
| 사용 기능   | 인텐트 파인더, 패스파인더, 클러스터 파인더                        |
| 구현 난이도 | 중                                                                |

### 엔진 2: 검색량/트렌드 수집 엔진 (Volume & Trend Collector)

| 항목        | 내용                                                                  |
| ----------- | --------------------------------------------------------------------- |
| 목적        | 각 키워드의 실제 월간 검색량, CPC, 트렌드 데이터 수집                 |
| 현재 상태   | `social-volume-collector.ts`, `trend-aggregator.ts` — 전체 시뮬레이션 |
| 필요한 변경 | Google Ads API + Google Trends + Naver DataLab 실제 연동              |
| 입력        | 키워드 목록                                                           |
| 출력        | 키워드별: 월간 검색량, CPC, 경쟁도, 12~48개월 트렌드, 성장률          |
| 사용 기능   | 인텐트 파인더 (핵심), 모든 기능 (보조)                                |
| 구현 난이도 | 중                                                                    |

### 엔진 3: SERP 분석 엔진 (SERP Analysis Engine)

| 항목        | 내용                                                                     |
| ----------- | ------------------------------------------------------------------------ |
| 목적        | Google 검색 결과 페이지를 분석하여 연관성, 도메인 경쟁, 콘텐츠 유형 파악 |
| 현재 상태   | **존재하지 않음**                                                        |
| 필요한 구현 | SERP 결과 수집 → 도메인 추출 → 키워드 간 결과 공유도 계산                |
| 입력        | 키워드 목록                                                              |
| 출력        | 키워드별 상위 10 URL, 도메인별 점유율, 키워드 간 SERP 유사도 행렬        |
| 사용 기능   | 클러스터 파인더 (핵심), 로드 뷰, 저니 파인더                             |
| 구현 난이도 | **상**                                                                   |

### 엔진 4: 검색 경로 빌더 (Search Path Builder)

| 항목        | 내용                                                                    |
| ----------- | ----------------------------------------------------------------------- |
| 목적        | 자동완성/연관 검색어 데이터로 before→target→after 검색 경로 그래프 구축 |
| 현재 상태   | `graph/graph-builder.ts` — 의도 기반 링크만 (실제 검색 순서 아님)       |
| 필요한 변경 | SERP 기반 순차적 검색 경로 구축, 최대 10단계                            |
| 입력        | SERP 데이터 (연관 검색어, 추천 검색어)                                  |
| 출력        | 방향성 그래프 (노드: 키워드, 엣지: 검색 순서 + 강도)                    |
| 사용 기능   | 패스파인더 (핵심), 로드 뷰                                              |
| 구현 난이도 | **상**                                                                  |

### 엔진 5: SERP 기반 클러스터링 엔진 (SERP Clustering Engine)

| 항목        | 내용                                                                |
| ----------- | ------------------------------------------------------------------- |
| 목적        | 검색 결과 공유도 기반으로 키워드를 의미 클러스터로 그룹핑           |
| 현재 상태   | `graph-builder.ts` 내 Louvain 변형 — 의도+시간구간 기반 (SERP 아님) |
| 필요한 변경 | SERP 유사도 행렬 기반 클러스터링으로 교체                           |
| 입력        | 키워드-SERP 유사도 행렬                                             |
| 출력        | 클러스터 목록 (키워드 그룹, 대표 키워드, 상위 도메인)               |
| 사용 기능   | 클러스터 파인더 (핵심)                                              |
| 구현 난이도 | 중 (SERP 데이터가 있으면)                                           |

### 엔진 6: 행동 패턴 기반 페르소나 엔진 (Behavioral Persona Engine)

| 항목        | 내용                                                              |
| ----------- | ----------------------------------------------------------------- |
| 목적        | 검색 여정 데이터에서 공통 행동 패턴을 추출하여 페르소나 자동 생성 |
| 현재 상태   | `persona/page.tsx` 내 — 5개 하드코딩 템플릿                       |
| 필요한 변경 | 검색 경로 군집 분석 → 행동 그룹 추출 → LLM 기반 레이블링          |
| 입력        | 검색 경로 그래프, 클러스터 데이터                                 |
| 출력        | 페르소나 그룹 (행동 패턴, JTBD, 대표 키워드, AI 인사이트)         |
| 사용 기능   | 페르소나 뷰 (핵심)                                                |
| 구현 난이도 | **상**                                                            |

### 엔진 7: 방향성 경로 분석 엔진 (Directed Path Analyzer)

| 항목        | 내용                                                          |
| ----------- | ------------------------------------------------------------- |
| 목적        | 시작 키워드→끝 키워드 사이의 구체적 검색 경로 추적            |
| 현재 상태   | **존재하지 않음**                                             |
| 필요한 구현 | 검색 경로 그래프에서 A→B 최단/최빈 경로 탐색 (BFS/다익스트라) |
| 입력        | 시작 키워드, 끝 키워드, 검색 경로 그래프                      |
| 출력        | 중간 단계 키워드 시퀀스, 각 단계의 전환 강도                  |
| 사용 기능   | 로드 뷰 (핵심)                                                |
| 구현 난이도 | 중 (경로 데이터가 있으면)                                     |

### 엔진 8: GPT 종합 분석 엔진 (GPT Analysis Engine)

| 항목        | 내용                                                |
| ----------- | --------------------------------------------------- |
| 목적        | 분석 결과를 전문가 수준으로 해석                    |
| 현재 상태   | 클러스터 분석만 구현됨                              |
| 필요한 변경 | 패스파인더, 페르소나, 로드 뷰 등 전체 기능으로 확장 |
| 입력        | 각 기능의 분석 결과 데이터                          |
| 출력        | 자연어 요약, 인사이트, 추천 행동                    |
| 사용 기능   | 전체                                                |
| 구현 난이도 | 중                                                  |

## 3. 필요한 저장 구조

### 현재 Prisma 모델 상태

| 모델                   | 상태          | 문제점                                                           |
| ---------------------- | ------------- | ---------------------------------------------------------------- |
| `IntentQuery`          | 스키마 정의됨 | tRPC 경로에서만 저장, REST 경로에서는 미저장                     |
| `IntentKeywordResult`  | 스키마 정의됨 | `searchVolume`, `socialVolume`, `trend` 등 핵심 필드가 항상 null |
| `SearchPath`           | 스키마 정의됨 | **DB에 실제 저장되지 않음** — 서비스에서 이 모델에 쓰지 않음     |
| `Persona`              | 스키마 정의됨 | **DB에 실제 저장되지 않음**                                      |
| `PersonaKeyword`       | 스키마 정의됨 | **DB에 실제 저장되지 않음**                                      |
| `KeywordClusterResult` | 스키마 정의됨 | **DB에 실제 저장되지 않음**                                      |
| `BrandJourney`         | 스키마 정의됨 | **DB에 실제 저장되지 않음**                                      |

### 추가 필요한 저장 구조

| 모델/테이블          | 목적                                                    | 우선순위 |
| -------------------- | ------------------------------------------------------- | -------- |
| `SerpResult`         | SERP 수집 결과 캐시 (키워드별 상위 URL, 도메인, 수집일) | 높음     |
| `KeywordVolume`      | 월간 검색량 히스토리 (키워드, 연월, 검색량, CPC)        | 높음     |
| `TrendData`          | 트렌드 시계열 데이터 (키워드, 월별 관심도)              | 중간     |
| `DomainVisibility`   | 도메인별 SERP 점유율 (도메인, 키워드, 순위, 스냅샷일)   | 중간     |
| `SearchPathSequence` | 검색 경로 시퀀스 (경로ID, 단계 번호, 키워드, 전환 강도) | 중간     |
| `ConsumerJourney`    | CDJ 단계별 키워드 매핑 (저니 파인더용)                  | 낮음     |

### DB 마이그레이션 현황

- Prisma 스키마 + 수동 SQL 파일은 작성 완료
- **PostgreSQL이 아직 실행되지 않아 마이그레이션 미적용**
- 5개 테이블(SearchPath, Persona, PersonaKeyword, KeywordClusterResult, BrandJourney)이 아직 실제 DB에 없음

## 4. 데이터 흐름 아키텍처 (목표 상태)

```
[사용자 입력: 키워드]
      │
      ▼
┌─────────────────────────┐
│  키워드 확장 엔진        │ ← Google Autocomplete + Naver 자동완성
│  (실제 API 연동)         │ ← Google Related Searches
└─────────┬───────────────┘
          │ 확장된 키워드 목록 (최대 20K)
          ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│  검색량/트렌드 수집      │     │  SERP 분석 엔진          │
│  Google Ads + Trends     │     │  SerpAPI / DataForSEO    │
│  Naver DataLab           │     │  → 상위 URL, 도메인       │
└─────────┬───────────────┘     └─────────┬───────────────┘
          │                               │
          ▼                               ▼
┌─────────────────────────┐     ┌─────────────────────────┐
│  의도 분류 엔진          │     │  SERP 기반 클러스터링    │
│  규칙 + LLM (I/N/C/T)   │     │  결과 공유도 기반 그룹핑  │
└─────────┬───────────────┘     └─────────┬───────────────┘
          │                               │
          ▼                               ▼
┌─────────────────────────────────────────────────────────┐
│  통합 그래프 빌더                                        │
│  → 검색 경로 그래프 (패스파인더)                          │
│  → 키워드 클러스터 (클러스터 파인더)                      │
│  → 콘텐츠 갭 매트릭스                                    │
│  → 페르소나 행동 패턴 추출                                │
└─────────┬───────────────────────────────┬───────────────┘
          │                               │
          ▼                               ▼
┌─────────────────────┐         ┌─────────────────────────┐
│  DB 저장             │         │  GPT 종합 분석           │
│  (Prisma 모델)       │         │  → 전체 기능 커버         │
└─────────────────────┘         └─────────────────────────┘
          │
          ▼
     [프론트엔드 시각화]
```
