# ListeningMind Gap Analysis

## X2가 리스닝마인드 대비 부족한 부분

---

### 1. 기능 Gap

| Gap | 심각도 | 설명 |
|-----|--------|------|
| **인구통계 분석 부재** | HIGH | 성별/연령/세대 데이터가 코드베이스 전체에 없음. 리스닝마인드는 인구통계 기반 세분화를 제공하나 X2는 행동 기반 페르소나만 존재 |
| **카테고리 엔트리 포인트** | MEDIUM | 특정 카테고리로 진입하는 검색 경로 분석 없음. 키워드→카테고리 매핑 체계 없음 |
| **다중 키워드 동시 입력** | LOW | 현재 seed 1개→자동 확장. 사용자가 5-10개 키워드를 직접 입력하여 비교하는 UI 없음 |
| **클러스터 네트워크 시각화** | MEDIUM | 클러스터 결과를 네트워크/방사형 그래프로 보여주지 않음. 리스트+바차트만 |
| **LLM 기반 인사이트 해석** | MEDIUM | "Upgrade path" 주석만 있고 실제 LLM 해석 문장 생성이 없음. GPT 패널은 Cluster에만 존재 |

### 2. 데이터 Gap

| Gap | 심각도 | 설명 |
|-----|--------|------|
| **절대 검색량** | MEDIUM | Google Ads API 키 미설정 시 상대값(0-100)만. 리스닝마인드는 절대 검색량 표시 |
| **인구통계 데이터 소스** | HIGH | 네이버 DataLab, Google Trends 모두 인구통계를 제공하지 않음. 별도 소스 필요 |
| **실데이터 수집 전제조건** | MEDIUM | Worker + Redis + API 키 3가지가 모두 필요. 하나라도 없으면 silent fail (hasData: false) |

### 3. UX Gap

| Gap | 심각도 | 설명 |
|-----|--------|------|
| **워드클라우드** | LOW | RelatedKeywordMap이 크기맵 역할을 하나 전통적 워드클라우드 기대와 다름 |
| **실시간 신규 키워드 탐지** | MEDIUM | 트렌드 차트 표시만 되고, 새로 등장한 키워드를 자동 감지하여 알려주는 기능 없음 |
| **카테고리 네비게이션** | LOW | 업종(BEAUTY/FNB 등) 선택은 있으나 검색 카테고리 트리 탐색 UX 없음 |

### 4. 운영 Gap

| Gap | 심각도 | 설명 |
|-----|--------|------|
| **Zero-config 초기 경험** | MEDIUM | API 키 설정 + Worker 실행 없이는 모든 분석 결과가 비어 있음. 온보딩 가이드 없음 |
| **데이터 수집 모니터링** | LOW | Worker 상태, 수집 성공/실패 현황을 보여주는 admin UI 없음 |
