# X2 Data Source Portfolio Strategy

## 국내 데이터 축

| 소스 | 수집 방식 | 데이터 유형 | 상태 |
|------|----------|-----------|------|
| 네이버 블로그/뉴스/카페/지식iN | **공식 API** | 검색 결과 + 원문 발췌 | 완료 |
| 네이버 DataLab | **공식 API** | 검색 트렌드 (상대값) + 성별/연령 | 완료 |
| 네이버 웹문서 | **공식 API** | 웹문서 검색 결과 | 미착수 (같은 키 사용) |
| YouTube (한국) | **공식 API** | 댓글/영상/채널 메타 | 키 설정 필요 |

## 글로벌 데이터 축

| 소스 | 수집 방식 | 데이터 유형 | 상태 |
|------|----------|-----------|------|
| GDELT | **공개 API (무료)** | 65개 언어 글로벌 뉴스, 국가/감성 | 구현 완료 |
| NewsAPI | **공식 API** | 80,000+ 뉴스 소스, 언어/국가 필터 | 완료 |
| SerpAPI | **공식 API** | Google SERP/Trends/Related | 완료 |
| X (Twitter) | **공식 API** | 실시간 트윗, 최근 7일 | 키 설정 필요 |
| Reddit | **공식 API (무료)** | 글로벌 커뮤니티, 서브레딧별 | 미착수 |

## 사용자 연결형 데이터 축

| 소스 | 수집 방식 | 데이터 유형 | 상태 |
|------|----------|-----------|------|
| 고객 YouTube 채널 | **사용자 OAuth** | 1st-party 채널 분석 | 채널 연동 구조 있음 |
| 고객 Instagram | **사용자 OAuth** | 비즈니스 계정 인사이트 | 어댑터 있음, 키 필요 |
| 고객 TikTok | **사용자 연동** | 비즈니스 계정 데이터 | 어댑터 있음, 키 필요 |
| CSV/데이터 업로드 | **사용자 직접 입력** | CRM/리뷰/CS 데이터 | 미착수 |

## 소스별 수집 전략

```
1. 공식 API (최우선)
   → 네이버, YouTube, SerpAPI, NewsAPI, GDELT, X, Instagram, TikTok

2. 공개 데이터 (무료)
   → GDELT, Wikipedia, 정부 공공데이터

3. 사용자 연결 (OAuth)
   → 고객 채널 연동, 비즈니스 계정

4. 제휴/라이선스 (유료)
   → DataForSEO, Google Ads (Standard Access)

5. 크롤링 (최후 수단, 법적 검토 후)
   → 국내 커뮤니티 (공식 API 없는 소스만)
```
