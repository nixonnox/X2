## 1순위: 랜딩 페이지 (비로그인)

**파일**: `apps/web/src/app/page.tsx` (22줄)

### 현재 상태

- 단 3개 요소만 존재: "X2" 로고 텍스트, 영어 설명문, 로그인 버튼
- 최소한의 placeholder 수준. 제품 설명, 가치 제안, 기능 소개 없음

### 가장 큰 문제

- **영어 텍스트**: "Social Media Analytics & Listening Platform"이 사용자에게 바로 노출
- 제품이 무엇인지, 왜 사용해야 하는지 전혀 전달하지 못함

### UX 가이드라인 위반

- [ ] 해요체 미적용 (설명 텍스트가 영어)
- [ ] 행동 중심 CTA 부재 (단순 "로그인" 버튼만 있음)
- [ ] 가치 제안(value proposition) 없음

### 디자인 가이드라인 위반

- [ ] `bg-indigo-600` 하드코딩 색상 사용 (토큰 미사용)
- [ ] `text-gray-500` 하드코딩 색상 사용
- [ ] 레이아웃 계층 구조 없음 (센터 정렬 텍스트만)

### 카피/한국어 이슈

- "Social Media Analytics & Listening Platform" -> 한국어 필요
- 로그인 링크가 `/api/dev-login`으로 개발용 경로 노출

### 필수 수정 항목

1. 영어 설명문을 한국어로 교체
2. 제품 핵심 가치 제안 3~4가지 추가
3. 주요 기능 시각적 미리보기 또는 스크린샷
4. CTA 버튼 문구 개선 ("무료로 시작하기" 등)
5. `/api/dev-login` 경로를 `/login`으로 변경
6. 하드코딩 색상을 CSS 변수로 교체

### 완료 기준

- 모든 텍스트 한국어 해요체
- CSS 변수 사용
- 3초 내 가치 전달 가능한 구조

### 예상 복잡도: **M** (중간)

## 2순위: 로그인 후 대시보드

**파일**: `apps/web/src/app/(dashboard)/dashboard/page.tsx` (130줄) + `dashboard-view.tsx` (1006줄)

### 현재 상태

- tRPC를 통해 실제 데이터를 가져옴 (채널 요약, KPI, 콘텐츠)
- 채널이 없을 때 온보딩 가이드 + 채널 등록 유도 존재
- 채널이 있을 때 KPI 카드 + 트렌드 차트 + 최근 콘텐츠 + 서치 인텔리전스 상태바 표시
- 추천 액션 영역이 아직 placeholder 상태 (Phase 9 예정)

### 가장 큰 문제

- **1006줄 단일 파일**에 너무 많은 컴포넌트가 인라인으로 정의됨
- 추천 액션(RecommendedActions)이 하드코딩된 목업 상태
- 서치 인텔리전스 카드들이 데이터 없을 때의 처리가 복잡하고 일관성 없음

### UX 가이드라인 위반

- [ ] `"분석이 완료되면 추천 액션이 여기에 표시됩니다"` - "~합니다"체 (해요체 아님)
- [ ] `"채널을 등록하면 추천 액션이 표시됩니다"` - 동일
- [ ] 일부 설명에 "~합니다"와 "~해요" 혼용

### 디자인 가이드라인 위반

- [ ] `bg-gray-100`, `text-gray-500`, `text-red-600` 등 하드코딩 색상 다수
- [ ] `text-blue-600` 하드코딩 링크 색상
- [ ] KPI 영역에 `KpiCard` 공유 컴포넌트 사용하지만, 내부 카드들은 자체 스타일
- [ ] `rounded-xl` 일부 사용 (표준: `rounded-lg`)

### 카피/한국어 이슈

- `// TODO: Populate from channel snapshots when historical data exists` - 영어 TODO
- "~합니다" 체와 "~해요" 체 혼용

### 필수 수정 항목

1. dashboard-view.tsx를 여러 컴포넌트 파일로 분리
2. 모든 "~합니다"를 "~해요"로 통일
3. 추천 액션 영역을 실제 API와 연결하거나, 의미 있는 빈 상태로 교체
4. 하드코딩 색상을 CSS 변수로 교체
5. `rounded-xl` -> `rounded-lg` 통일
6. TODO 코멘트 정리

### 완료 기준

- 파일 크기 500줄 이하 (나머지 컴포넌트 분리)
- 모든 텍스트 해요체
- CSS 변수 사용
- 채널 유/무 모두 의미 있는 화면

### 예상 복잡도: **L** (대형)

## 3순위: Intelligence Hub (핵심 분석 페이지)

**파일**: `apps/web/src/app/(dashboard)/intelligence/page.tsx` (1292줄)

### 현재 상태

- 프로젝트에서 가장 큰 단일 페이지 파일 (1292줄)
- 키워드 입력 -> AI 분석 -> 다차원 결과 표시 구조
- 26개의 임포트된 컴포넌트 사용
- 실제 tRPC API와 연동되어 동작

### 가장 큰 문제

- **영어 제목**: "Intelligence Hub" (사용자 노출)
- 1292줄 단일 파일에 페이지 전체 로직이 집중
- 화려한 그라데이션 헤더(`bg-gradient-to-br from-violet-600 via-violet-500 to-indigo-600`)가 다른 페이지와 톤 불일치

### UX 가이드라인 위반

- [ ] "Intelligence Hub" 영어 제목
- [ ] `Auto-refresh every 60s` 영어 코멘트 (UI 미노출이지만 코드 일관성)
- [ ] PageHeader 컴포넌트 미사용 (자체 그라데이션 헤더)

### 디자인 가이드라인 위반

- [ ] `rounded-2xl` 사용 (표준 위반)
- [ ] `rounded-xl` 다수 사용
- [ ] 그라데이션 배경이 전체 디자인 시스템과 불일치
- [ ] `text-gray-700`, `text-gray-900` 등 하드코딩 색상
- [ ] `shadow-lg shadow-violet-200/40` 특수 그림자
- [ ] `space-y-2`~`space-y-6` 혼용

### 카피/한국어 이슈

- "Intelligence Hub" -> "인텔리전스 허브" 또는 "통합 분석"
- "풍부해요" / "보통이에요" / "최소한이에요" - 해요체는 맞으나 맥락에 맞는지 검토 필요

### 필수 수정 항목

1. 페이지 제목 한국어화
2. 그라데이션 헤더를 일반 PageHeader로 교체
3. 파일 분리 (최소 3개 이상 컴포넌트 파일)
4. `rounded-2xl`, `rounded-xl` -> `rounded-lg`
5. 하드코딩 색상을 CSS 변수로 교체
6. 섹션 간격 `space-y-6`으로 통일

### 완료 기준

- 파일 크기 500줄 이하
- PageHeader 컴포넌트 사용
- 모든 텍스트 한국어
- CSS 변수 사용
- 전체 디자인 시스템과 시각적 일관성

### 예상 복잡도: **L** (대형)

## 4순위: 채널 관리

- [ ] 채널 목록의 border 스타일이 일부 직접 정의

### 카피/한국어 이슈

- `searchPlaceholder`가 i18n으로 관리됨 (양호)
- 에러 메시지 내 영어 비교 문자열

### 필수 수정 항목

1. 채널 등록 페이지 파일 분리 (스텝별 컴포넌트)
2. ChannelKpiCard -> 공유 KpiCard 또는 StatCard로 통일
3. 에러 메시지 한국어 처리

### 완료 기준

- KPI 카드 스타일 통일

## 5순위: Listening Hub (통합 리스닝)

**파일**: `apps/web/src/app/(dashboard)/listening-hub/page.tsx` + 8개 섹션 컴포넌트

### 현재 상태

- 시드 키워드 입력 -> `/api/search-intelligence/analyze` API 호출
- 결과를 IntentSummary, Cluster, Pathfinder, RoadView, Persona, Insight, Action, Evidence 8개 섹션으로 표시
- 컴포넌트 분리가 비교적 잘 되어 있음

### 가장 큰 문제

- 전체 분석이 단일 API 호출로 처리되어, 부분 실패 시 전체가 에러 상태가 됨
- 세션 만료 처리가 하드코딩 (`res.type === "opaqueredirect"`)

### UX 가이드라인 위반

- [ ] 세션 만료 메시지: "세션이 만료되었습니다. 페이지를 새로고침 후 다시 시도해주세요." - "~습니다" 체
- [ ] PageHeader 사용하지만 자체 검색 바 스타일

### 디자인 가이드라인 위반

- [ ] 페이지 레벨 레이아웃이 섹션 컴포넌트에 위임되어 일관성 불확실
- [ ] `rounded-md` 검색 입력

### 카피/한국어 이슈

- "세션이 만료되었습니다" -> "세션이 만료됐어요"
- "페이지를 새로고침 후 다시 시도해주세요" -> "다시 로그인해주세요"

### 필수 수정 항목

1. 에러/세션 만료 메시지 해요체로 통일
2. 부분 실패 처리 개선 (일부 성공한 섹션은 표시)
3. 각 섹션 컴포넌트의 디자인 일관성 점검

### 완료 기준

- 모든 텍스트 해요체
- 부분 실패 시에도 가능한 데이터 표시
- 8개 섹션 컴포넌트 디자인 통일

### 예상 복잡도: **M** (중간)
