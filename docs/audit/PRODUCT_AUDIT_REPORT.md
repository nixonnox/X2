# X2 제품 종합 감사 보고서

> 작성일: 2026-03-09
> 대상: X2 SNS 채널 분석 SaaS (현재 MVP 단계)

---

## 1. 전체 요약

X2는 YouTube/Instagram/TikTok/X 채널을 분석하는 SaaS 플랫폼이다.
현재 **UI 프로토타입 수준**으로, 대부분의 화면이 목업 데이터에 의존하고 있다.
실제 데이터 수집 파이프라인은 설계되었으나 연결되지 않은 상태이다.

### 핵심 수치

- 전체 페이지: **38개** (마케팅 1, 인증 6, 대시보드 22, 관리자 8, 공개 1)
- 컴포넌트: **57개**
- API 라우트: **13개**
- DB 모델: **23개** (정의됨, 대부분 미사용)
- lib 파일: **106개**

### 심각도별 문제 분류

| 심각도 | 항목 수 | 설명                         |
| ------ | ------- | ---------------------------- |
| 치명적 | 5       | 핵심 기능 미작동             |
| 높음   | 8       | 기능은 있으나 실질 동작 안함 |
| 보통   | 12      | UX 개선 필요                 |
| 낮음   | 6       | 편의성/완성도                |

## 2. 기능별 상세 점검

### 2.1 랜딩/소개/데모 진입

| 항목        | 상태 | 설명                                 |
| ----------- | ---- | ------------------------------------ |
| 랜딩 페이지 | 존재 | `(marketing)/page.tsx` — 기본 구조만 |
| 서비스 소개 | 미흡 | 기능 설명, 스크린샷, 가격 비교 없음  |
| 데모 체험   | 없음 | 비로그인 데모 불가, 바로 가입 요구   |
| 가격 안내   | 없음 | 랜딩에 요금제 비교 없음              |

**문제점:**

- 첫 방문자가 서비스를 이해하고 가입할 동기가 부족
- 데모 없이 가입해야 해서 진입 장벽 높음

### 2.2 회원가입/로그인/계정 복구

| 항목            | 상태   | 설명                                                 |
| --------------- | ------ | ---------------------------------------------------- |
| 로그인          | 작동   | Google OAuth + 개발모드 이메일 로그인                |
| 회원가입        | 존재   | `signup/page.tsx` 있으나 실제 가입 흐름 미검증       |
| 비밀번호 찾기   | 화면만 | `forgot-password/page.tsx` — 이메일 발송 미구현      |
| 비밀번호 재설정 | 화면만 | `reset-password/[token]/page.tsx` — 토큰 검증 미구현 |
| 이메일 인증     | 화면만 | `verify-email/[token]/page.tsx` — 실제 인증 미구현   |
| 온보딩          | 존재   | `onboarding/page.tsx` — 4단계 위저드 있음            |

**문제점:**

- 이메일 발송 시스템 미구현 (Mock만 존재)
- 비밀번호 찾기/재설정 흐름이 실제로 동작하지 않음
- 가입 후 온보딩 → 대시보드 전환 흐름 미검증

### 2.3 대시보드

| 항목           | 상태      | 설명                                |
| -------------- | --------- | ----------------------------------- |
| 메인 대시보드  | 화면 있음 | KPI 카드, 차트, 온보딩 가이드 표시  |
| KPI 데이터     | 100% 목업 | `DASHBOARD_KPIS` 하드코딩           |
| 채널 성과 차트 | 100% 목업 | `CHANNEL_PERFORMANCE_DATA` 하드코딩 |
| 트렌딩 키워드  | 100% 목업 | `TRENDING_KEYWORDS` 하드코딩        |
| 실시간 알림    | 없음      | 알림 시스템 미구현                  |

**문제점:**

- 대시보드가 등록된 채널 데이터를 전혀 반영하지 않음
- 사용자가 채널을 등록해도 대시보드 숫자가 변하지 않음
- 온보딩 가이드의 "완료" 상태가 실제 행동과 연결되지 않음

### 2.4 채널 관리

| 항목          | 상태      | 설명                                         |
| ------------- | --------- | -------------------------------------------- |
| 채널 목록     | 작동      | 필터링/검색 작동, 인메모리 스토어            |
| 채널 등록     | 작동      | URL 입력 → 플랫폼 감지 → 저장                |
| 채널 상세     | 부분 작동 | 목업 채널은 데이터 표시, 신규 채널은 빈 화면 |
| 채널 삭제     | 작동      | 인메모리에서 제거                            |
| 채널 수정     | 없음      | 수정 버튼 있으나 기능 없음                   |
| 데이터 동기화 | 부분      | API 키 있으면 작동, 없으면 에러 반환         |

**문제점:**

- 채널 등록 후 데이터가 자동으로 수집되지 않음 (수동 동기화 필요)
- API 키 미설정 시 사용자에게 설정 방법 안내 없음
- 채널 수정 기능 미구현
- 인메모리 스토어라서 서버 재시작 시 데이터 소실

## 4. 우선순위 정의

### P0 — 치명적 (제품 핵심 기능 불가)

1. **DB 연결 없음**: PostgreSQL 미실행, Prisma 미생성 → 모든 데이터가 휘발성
2. **채널 데이터 수집 미연결**: 등록 후 데이터 없음, 사용자 이탈 원인
3. **대시보드가 실제 데이터를 반영하지 않음**: 항상 같은 숫자
4. **댓글 수집 파이프라인 없음**: 핵심 기능인 댓글 분석 불가
5. **결제/플랜 제한 없음**: 수익 모델 미작동

### P1 — 높음 (핵심 흐름 끊김)

6. **이메일 시스템 미구현**: 비밀번호 찾기, 리포트 발송 불가
7. **AI 실제 프로바이더 미연결**: Mock만 사용
8. **경쟁 채널 저장 미구현**: 폼 제출 후 저장 안됨
9. **키워드 등록/수집 미구현**: Intent 엔진은 있으나 UI 미연결
10. **관리자 권한 체크 없음**: 보안 취약
11. **리포트 생성 실제 미동작**: AI 호출 없이 목업만
12. **채널 수정 기능 없음**: 등록 후 수정 불가
