# X2 프로세스 흐름 점검 보고서

> 작성일: 2026-03-09

## 1. 핵심 사용자 흐름 점검

### 흐름 1: 첫 방문 → 서비스 이해 → 데모 체험

```
[현재 흐름]
첫 방문 → 랜딩(마케팅) 페이지 → ??? → 회원가입

[문제점]
- 랜딩 페이지에 서비스 설명/스크린샷/가격 비교가 없음
- "무료로 시작하기" 또는 "데모 보기" 버튼 없음
- 비회원은 대시보드를 볼 수 없어서 서비스 가치 판단 불가
- dead-end: 랜딩 → 가입 외에 갈 곳이 없음

[개선 방향]
랜딩 → 기능 소개 섹션 → 스크린샷/데모 → 요금제 비교 → 가입 CTA
                                          ↘ 데모 체험(로그인 없이)
```

### 흐름 2: 회원가입 → 로그인 → 온보딩

```
[현재 흐름]
가입 페이지 → (Google OAuth 또는 이메일) → 로그인 → 대시보드(또는 온보딩?)

[문제점]
- 가입 성공 후 자동 로그인 여부 불명확
- 가입 → 온보딩 자동 리다이렉트 여부 불명확
- 온보딩 완료 후 대시보드로 가는 흐름 미검증
- 이메일 인증 흐름이 실제로 동작하지 않음 (이메일 발송 없음)
- 가입 실패 시 에러 메시지 불충분

[끊긴 지점]
✗ 가입 → 이메일 인증 (이메일 발송 미구현)
✗ 온보딩 4단계 완료 → 각 단계가 실제 기능과 연결되지 않음
  - "채널 추가" 단계: 실제 채널 추가 페이지로 이동하는지 불명확
  - "키워드 등록" 단계: 키워드 등록 기능 자체가 없음
  - "경쟁 채널 추가" 단계: 경쟁 채널 저장 미구현
```

### 흐름 3: 로그인 후 첫 화면 진입

```
[현재 흐름]
로그인 → /home → /dashboard 리다이렉트

[문제점]
- 첫 로그인 사용자와 기존 사용자를 구분하지 않음
- 채널이 0개인 사용자에게도 목업 KPI가 표시됨 → 혼란
- "시작하기" 가이드가 있지만 실제 동작과 연결 약함
- 대시보드의 모든 숫자가 사용자 데이터가 아닌 목업

[개선 방향]
첫 로그인 → 온보딩 위저드 (채널 1개 등록 유도)
기존 사용자 → 실제 채널 데이터 기반 대시보드
채널 0개 → Empty State + "첫 채널 등록하기" CTA
```

### 흐름 4: 채널 등록 → 채널 분석

```
[현재 흐름]
채널 목록 → "채널 추가" → 등록 폼 → 저장 → 채널 목록으로 이동
→ 채널 클릭 → 상세 페이지 → "데이터 수집 중" 배너

[문제점]
1. 채널 등록 후 자동 데이터 수집이 시작되지 않음
2. "지금 데이터 수집하기" 버튼 → API 키 미설정 에러
   → 사용자에게 API 키 설정 방법을 안내하지 않음
3. 데이터 수집 성공해도 시계열이 1개 포인트라 차트가 무의미
4. "분석 재실행" 버튼이 있으나 기능 없음
5. "수정" 버튼이 있으나 기능 없음

[끊긴 지점]
✗ 채널 등록 → 자동 데이터 수집 (수동만 가능)
✗ 데이터 수집 실패 → 사용자 안내 부족
✗ 채널 상세 → 댓글 분석 CTA → 댓글 페이지 (댓글 데이터 없음)
✗ 채널 상세 → 경쟁 분석 CTA → 경쟁 페이지 (경쟁 데이터 없음)
```

### 흐름 5: 댓글 수집 → 댓글 분석 → 인사이트

```
[현재 흐름]
사이드바 "댓글 분석" → 댓글 대시보드 (목업 데이터)
또는
채널 상세 → "댓글 분석 보기" CTA → 댓글 페이지

[문제점]
1. 댓글을 수집하는 파이프라인이 없음 (플랫폼 API 호출 안함)
2. 모든 댓글 데이터가 하드코딩
3. AI 감성 분석이 연결되지 않음
4. 댓글 → 인사이트 → 액션 흐름이 끊김
5. 채널별 댓글 필터링이 실제 데이터와 연결되지 않음

[끊긴 지점]
✗ 콘텐츠 → 댓글 수집 (파이프라인 없음)
✗ 댓글 → AI 감성 분석 (Mock만)
✗ 분석 결과 → 인사이트 생성 (연결 없음)
✗ 인사이트 → 추천 액션 (연결 없음)
```

### 흐름 6: 경쟁 채널 등록 → 비교 분석 → 추천 전략

```
[현재 흐름]
경쟁 분석 → 경쟁 채널 목록 (목업) → "추가" → 등록 폼
                                   → "비교" → 비교 페이지 (목업)

[문제점]
1. 경쟁 채널 추가 폼 제출이 실제로 저장되지 않음
2. 비교 분석 데이터가 완전히 하드코딩
3. 내 채널과 경쟁 채널의 실제 데이터 비교가 불가
4. AI 기반 전략 추천이 연결되지 않음

[끊긴 지점]
✗ 경쟁 채널 추가 → 저장 (미구현)
✗ 경쟁 채널 → 데이터 수집 (연결 없음)
✗ 비교 데이터 → 전략 추천 (AI 미연결)
```

### 흐름 7: 키워드 등록 → 리스닝 수집 → 트렌드 분석 → 인사이트

```
[현재 흐름]
사이드바 "키워드/트렌드" → 키워드 페이지 (목업 트렌드)
→ "멘션" 탭 → 멘션 분석 (목업)

[문제점]
1. 키워드 등록 기능 자체가 없음
2. Intent 분석 엔진(/lib/intent-engine/)이 잘 구현되어 있으나
   키워드 페이지에 연결되지 않음
3. 멘션 수집 파이프라인 없음
4. API 라우트(/api/intent/analyze)는 존재하지만 UI에서 호출하지 않음

[끊긴 지점]
✗ 키워드 등록 UI (없음)
✗ 키워드 → Intent 엔진 호출 (미연결)
✗ 멘션 수집 → 트렌드 분석 (파이프라인 없음)
✗ 트렌드 → 인사이트 (연결 없음)
```

### 흐름 8: 리포트 생성 → 예약 → 이메일/공유 링크

```
[현재 흐름]
인사이트 → 리포트 → 리포트 목록 (목업)
→ "리포트 만들기" → 생성 폼
→ 리포트 상세 → 이메일 발송 다이얼로그 / 공유 링크

[문제점]
1. 리포트 생성이 실제로 AI를 호출하여 내용을 만들지 않음
2. 리포트가 DB에 저장되지 않음
3. 이메일 발송이 Mock (실제 이메일 안감)
4. 스케줄러가 실제로 작동하지 않음 (BullMQ 미연결)
5. 공유 링크 페이지가 목업 데이터 표시

[끊긴 지점]
✗ 리포트 생성 → AI 호출 → 내용 생성 (Mock만)
✗ 이메일 발송 (Mock 프로바이더)
✗ 스케줄 → 자동 생성 (BullMQ 없음)
✗ 공유 링크 → 실제 리포트 (목업)
```

### 흐름 9: 무료회원 → 업그레이드 → 결제 → 유료 기능 사용

```
[현재 흐름]
빌링 페이지 → 요금제 목록 → "업그레이드" 버튼 → ???

[문제점]
1. 결제 버튼이 Stripe Checkout으로 연결되지 않음
2. 현재 플랜 표시 없음 (사용자가 어떤 플랜인지 모름)
3. 플랜별 기능 제한 적용 없음
4. 무료 → 유료 전환 시 추가 기능 활성화 없음
5. 결제 성공/실패 처리 없음

[끊긴 지점]
✗ 업그레이드 버튼 → Stripe Checkout (미구현)
✗ 결제 성공 → 플랜 변경 (웹훅 없음)
✗ 플랜 → 기능 제한 (체크 없음)
```

### 흐름 10: 관리자 → 수집/AI/플랜 상태 확인

```
[현재 흐름]
사이드바 "관리" → AI 관리 / 데이터 수집 / 사용자 관리 등

[문제점]
1. 관리자 권한 체크 없음 → 일반 사용자도 접근 가능
2. 대부분 스텁 페이지 ("개발 중")
3. AI 관리 데이터도 목업
4. 데이터 수집 관리의 커넥터 상태가 실제가 아님

[끊긴 지점]
✗ 관리자 인증/권한 (없음)
✗ 수집 상태 → 실제 커넥터 상태 (목업)
✗ AI 상태 → 실제 사용량/비용 (목업)
```

## 2. 예외 흐름 점검

### 네트워크/API 에러

| 상황                   | 현재 처리             | 개선 필요                         |
| ---------------------- | --------------------- | --------------------------------- |
| API 호출 실패          | 글로벌 error.tsx 표시 | 인라인 에러 + 재시도 버튼         |
| 플랫폼 API 키 미설정   | 에러 메시지 반환      | 설정 방법 안내 + 설정 페이지 링크 |
| 플랫폼 API 할당량 초과 | 에러 메시지 반환      | 남은 시간 안내                    |
| 채널 찾을 수 없음      | "not found"           | 올바른 URL 형식 안내              |
| 데이터 수집 실패       | 에러 메시지           | 재시도 옵션 + 실패 원인 설명      |

### 인증/권한 에러

| 상황                             | 현재 처리         | 개선 필요                 |
| -------------------------------- | ----------------- | ------------------------- |
| 세션 만료                        | /login 리다이렉트 | callbackUrl 보존 (구현됨) |
| OAuth 실패                       | Auth.js 기본 에러 | 커스텀 에러 페이지        |
| 관리자 아닌 사용자가 /admin 접근 | 허용됨            | 403 또는 리다이렉트       |
| 플랜 제한 초과                   | 허용됨            | 업그레이드 유도           |

### 데이터 경계 상황

| 상황             | 현재 처리             | 개선 필요            |
| ---------------- | --------------------- | -------------------- |
| 채널 0개         | 목업 데이터 표시      | Empty State + CTA    |
| 데이터 수집 중   | "데이터 수집 중" 배너 | 진행률 표시          |
| 데이터 없는 차트 | 차트 숨김             | "데이터 부족" 메시지 |
| 중복 채널 등록   | 409 에러 반환         | 기존 채널 링크 제공  |

## 4. 전체 흐름 시각화

```
[방문자]
  │
  ├→ 랜딩 ─── ✗ 서비스 소개 부족
  │     │
  │     └→ 회원가입 ─── ✗ 이메일 인증 미작동
  │           │
  │           └→ 로그인 ── 작동
  │                 │
  │                 └→ 온보딩 ─── ✗ 단계별 실제 기능 미연결
  │                       │
  │                       └→ 대시보드 ─── ✗ 목업 데이터만
  │
[로그인 사용자]
  │
  ├→ 채널 등록 ─── 작동 ──→ 채널 목록 ── 작동
  │     │                        │
  │     └→ 데이터 수집 ─── ✗ API 키 필요, 자동 수집 없음
  │           │
  │           └→ 채널 분석 ─── 부분 작동 (신규 채널은 빈 화면)
  │
  ├→ 댓글 분석 ─── ✗ 수집 파이프라인 없음, 100% 목업
  │
  ├→ 경쟁 분석 ─── ✗ 저장 미구현, 100% 목업
  │
  ├→ 키워드/리스닝 ─── ✗ 등록/수집 미구현, Intent 엔진 미연결
  │
  ├→ 인사이트 ─── ✗ AI 미연결, 100% 목업
  │
  ├→ 리포트 ─── ✗ 생성/발송 미구현, 100% 목업
  │
  ├→ 결제 ─── ✗ Stripe 미연결
  │
  └→ 설정 ─── 부분 (언어 전환 작동)

[관리자]
  │
  └→ 관리 메뉴 ─── ✗ 권한 체크 없음, 대부분 스텁
```

## 5. 보완 우선순위

### 즉시 수정 (P0)

1. 채널 등록 → 자동 데이터 수집 트리거
2. 대시보드 → 등록된 채널 데이터 반영
3. DB 연결 또는 인메모리 → DB 전환 시작

### 단기 수정 (P1)

4. 댓글 수집 파이프라인 구현
5. 키워드 등록 → Intent 엔진 연결
6. 경쟁 채널 저장 구현
7. AI 실제 프로바이더 연결
8. 관리자 권한 체크 추가

### 중기 수정 (P2)

9. 이메일 시스템 연동 (Resend)
10. Stripe 결제 연동
11. 리포트 AI 생성 연결
12. 랜딩 페이지 보강
