## 1. 빠른 수정 (1-2일, 코드 변경 소규모)

### UX-001: 대시보드 Empty State

- **현재**: 채널 0개여도 목업 KPI가 표시됨 → 사용자 혼란
- **개선**: 채널이 없으면 "첫 채널을 등록해보세요" Empty State + CTA 버튼
- **파일**: `(dashboard)/dashboard/page.tsx`
- **난이도**: 낮음

### UX-002: 채널 등록 후 자동 수집 안내

- **현재**: 등록 후 빈 화면에 "데이터 수집 중" 배너만 표시
- **개선**: 등록 즉시 자동 sync 트리거 + 진행 상태 표시
- **파일**: `api/channels/route.ts`, `channel-detail-view.tsx`
- **난이도**: 낮음

### UX-003: 데이터 수집 실패 시 안내 개선

- **현재**: "API key not configured" 영문 에러만 표시
- **개선**: 한국어 안내 + 설정 방법 링크 + 환경변수명 표시
- **파일**: `channel-detail-view.tsx` (SyncButton)
- **난이도**: 낮음

### UX-004: 업로드 빈도 차트 Math.random() 제거

- **현재**: `Math.random()` 기반 가짜 데이터
- **개선**: 콘텐츠 publishedAt 기반 실제 빈도 계산, 데이터 없으면 차트 숨김
- **파일**: `channel-detail-view.tsx:74-77`
- **난이도**: 낮음

### UX-005: 토스트/알림 메시지 추가

- **현재**: 성공/실패 피드백이 인라인 텍스트만
- **개선**: 토스트 컴포넌트로 일관된 피드백 (채널 등록, 데이터 수집, 삭제 등)
- **파일**: 전역 (toast provider 필요)
- **난이도**: 낮음

### UX-006: 사이드바 현재 페이지 하이라이트

- **현재**: 현재 위치 표시가 약함
- **개선**: 활성 메뉴 항목 강조 + breadcrumb 추가
- **파일**: `components/layout/sidebar.tsx`
- **난이도**: 낮음

### UX-007: 하드코딩된 사용자 정보 제거

- **현재**: top-bar에 "U", "user@x2.local" 하드코딩

- **난이도**: 낮음

### UX-008: 에러 페이지 i18n

- **현재**: error.tsx에 한국어만 하드코딩

## 2. 구조 수정 (3-5일, 중규모 변경)

### UX-101: API 키 설정 페이지

- **현재**: API 키를 .env.local에 직접 입력해야 함
- **개선**: 설정 > API 키 관리 페이지에서 환경변수 상태 확인 + 설정 가이드
- **파일**: 신규 페이지 `settings/api-keys/page.tsx`
- **난이도**: 보통
- **설계**:
  - `/api/sync/status` API 호출로 현재 설정 상태 표시
  - 플랫폼별 "설정됨/미설정" 배지
  - 각 플랫폼 API 키 발급 가이드 링크
  - (보안상 키 값은 표시하지 않고 상태만 표시)

### UX-102: 관리자 메뉴 접근 제한

- **현재**: 모든 사용자가 /admin/\* 접근 가능
- **개선**: 세션에서 역할 확인 → OWNER/ADMIN만 메뉴 표시 + 접근 허용
- **파일**: `sidebar.tsx`, `middleware.ts`
- **난이도**: 보통

### UX-103: 채널 목록 Empty State

- **현재**: 채널 0개 시에도 필터 바가 표시됨
- **개선**: 채널 0개 → 간단한 소개 + "첫 채널 등록" 풀사이즈 CTA
- **파일**: `channels-list-view.tsx`
- **난이도**: 보통

### UX-104: 온보딩 → 실제 기능 연결

- **현재**: 온보딩 4단계가 설명만 표시
- **개선**: 각 단계에서 실제 해당 페이지로 이동하는 CTA
  - 1단계: 채널 등록 페이지로 이동
  - 2단계: (키워드 등록 기능 구현 후) 키워드 페이지로 이동
  - 3단계: 경쟁 채널 추가 페이지로 이동
  - 4단계: 대시보드로 이동
- **파일**: `onboarding/onboarding-wizard.tsx`
- **난이도**: 보통

### UX-105: 댓글 페이지 Empty State

- **현재**: 목업 데이터 표시
- **개선**: 댓글 수집 전 → "댓글 데이터가 아직 없습니다. 채널을 먼저 등록하고 데이터를 수집해주세요." + 채널 관리 링크
- **파일**: `comments/page.tsx`
- **난이도**: 보통

### UX-106: 경쟁 분석 저장 기능 연결

- **현재**: 경쟁 채널 추가 폼이 저장되지 않음
- **개선**: API 라우트 구현 + 인메모리 또는 DB 저장
- **파일**: `competitors/add/page.tsx`, 신규 API 라우트
- **난이도**: 보통

### UX-107: 키워드 등록 UI + Intent 엔진 연결

- **현재**: 키워드 등록 기능 없음, Intent 엔진 미연결
- **개선**: 키워드 입력 폼 + `/api/intent/analyze` 호출 + 결과 표시
- **파일**: `keywords/page.tsx`
- **난이도**: 보통~높음

### UX-108: 채널 상세 "분석 재실행" 버튼 연결

- **현재**: 버튼 있으나 기능 없음
- **개선**: sync API 호출 후 페이지 새로고침
- **파일**: `components/channels/channel-header.tsx`
- **난이도**: 낮음

### UX-109: 설정 페이지 언어 전환 개선

- **현재**: 동작하지만 UI가 불친절
- **개선**: 언어 선택 즉시 반영 + 확인 메시지
- **파일**: `settings/page.tsx`
- **난이도**: 낮음

### UX-110: 콘텐츠 페이지 채널별 필터

- **현재**: 전체 콘텐츠가 한 페이지에 표시
- **개선**: 채널 드롭다운 필터 + 날짜 범위 필터
- **파일**: `contents/page.tsx`
- **난이도**: 보통

## 3. 대규모 개선 (1-2주, 신규 기능/구조 변경)

### UX-201: 랜딩 페이지 전면 개편

- **현재**: 기본 구조만 있음
- **개선**:
  - 히어로 섹션: 핵심 가치 제안 + CTA
  - 기능 소개 섹션: 4대 기능 (채널 분석, 댓글 분석, 경쟁 분석, 리스닝)
  - 스크린샷/데모 GIF
  - 요금제 비교 테이블
  - 고객 후기/로고
  - FAQ
  - 푸터: 약관, 개인정보, 연락처
- **파일**: `(marketing)/page.tsx` + 신규 컴포넌트들
- **난이도**: 높음

### UX-202: 비회원 데모 모드

- **현재**: 로그인 없이는 아무것도 볼 수 없음
- **개선**: 목업 데이터 기반 "데모 보기" (읽기 전용, 등록/수정 불가)
- **파일**: 별도 라우트 또는 query param 기반
- **난이도**: 높음

### UX-203: 알림 시스템

- **현재**: 알림 벨 아이콘만 있고 기능 없음
- **개선**:
  - 데이터 수집 완료 알림
  - 구독자 변화 알림
  - 리스크 댓글 알림
  - AI 인사이트 생성 알림
- **파일**: 신규 시스템 (DB + UI + 실시간)
- **난이도**: 높음

### UX-204: 실시간 데이터 수집 진행 UI

- **현재**: 데이터 수집 시 "수집 중..." 텍스트만
- **개선**: 프로그레스 바 + 수집 단계 표시 (채널 정보 → 콘텐츠 → 스냅샷)
- **파일**: `channel-detail-view.tsx` (SyncButton), SSE 연결
- **난이도**: 보통~높음

### UX-205: 대시보드 실데이터 기반 재설계

- **현재**: 목업 KPI
- **개선**:
  - 등록된 채널 수, 총 구독자/팔로워, 총 콘텐츠
  - 최근 7일 참여율 변화
  - 리스크 댓글 건수
  - 미완료 추천 액션 수
  - 최근 수집 상태
- **파일**: `dashboard/page.tsx` + 서버 데이터 집계
- **난이도**: 높음

### UX-206: 모바일 반응형 최적화

- **현재**: 데스크톱 중심, 모바일에서 사이드바 겹침
- **개선**: 모바일 시 사이드바 숨김, 하단 탭바, 차트 축소
- **파일**: 레이아웃 + 각 페이지 반응형
- **난이도**: 높음

### UX-207: 다크모드 완성

- **현재**: Tailwind dark: 클래스 일부만 적용
- **개선**: 전체 페이지/컴포넌트 다크모드 테마 완성
- **파일**: 전체 CSS/컴포넌트
- **난이도**: 보통
