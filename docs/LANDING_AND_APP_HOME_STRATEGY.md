## Part 3: 설계 원칙

### 1. "요약 + 다음 행동" 우선

현재 대시보드는 숫자를 나열하지만, 사용자가 원하는 것은:

- "뭐가 달라졌나?" (변화 감지)
- "그래서 뭘 해야 하나?" (다음 행동)

**규칙**: 모든 숫자 옆에 변화율과 추천 행동을 함께 표시.

### 2. 복잡한 숫자 대시보드 지양

현재 대시보드의 6개 KPI는 모두 "총 XX" 형식의 누적 숫자. 사용자에게 인사이트를 주지 못함.

**규칙**:

- KPI 카드는 최대 4개
- 각 KPI에 반드시 변화율(+/-) 포함
- "총 조회수" 대신 "이번 주 조회수 변화" 같은 시간 기반 지표

### 3. 허브 진입점 명확화

현재 `/start` 페이지가 좋은 시도이나, 대시보드와 분리되어 있어 사용자가 둘 다 탐색해야 함.

**규칙**: 홈 = 대시보드 + 시작하기 통합. 별도 `/start` 페이지 불필요.

### 4. 점진적 복잡도 노출 (Progressive Disclosure)

현재 문제: 사이드바에 30+ 메뉴가 한 번에 노출 → 첫 방문자 압도.

**규칙**:

- 사이드바는 허브 레벨(4-5개)만 노출
- 하위 기능은 허브 진입 후 탭/서브메뉴로 노출
- "고급 기능"(인텔리전스 비교, 업종 프리뷰 등)은 분석 결과 내 "더 자세히 보기"로 연결
