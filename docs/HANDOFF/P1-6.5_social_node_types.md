# P1-6.5: @x2/social Node globals 타입 환경 추가

## 배경

P1-6 Part D에서 `@x2/social` 28건 잔존 확인. cold-review 가설(P1-3 pnpm overrides로 자동 해결)은 빗나갔고, 실제 원인은 ioredis 충돌이 아니라 **Node globals 타입 미선언**.

에러 분포 (28건):

- TS2304 19건: `fetch`, `URLSearchParams`, `URL`, `Response`, `setTimeout` 등
- TS2552 5건: `URLSearchParams` 유사어
- TS2580 4건: `process` (← @types/node 누락 신호)

파일별: instagram.ts 11, youtube.ts 8, tiktok.ts 5, x.ts 3, rate-limiter.ts 1.

## 진단

세 가지 사실의 결합:

1. `packages/config/typescript/library.json`의 `lib: ["ES2022"]` — DOM 없음. ES2022 표준 lib 자체는 fetch/URL/URLSearchParams/Response 정의 없음.
2. `packages/social/package.json`의 dependencies에는 `@x2/types`만 있음 → transitive 경로로 `@types/node`가 들어올 수 없음.
3. `packages/social/tsconfig.json`은 그냥 `extends: "@x2/tsconfig/library.json"`뿐이라 `types` 옵션도 없음.

다른 패키지(analyzer 등)가 멀쩡했던 이유는 `ioredis`/`bullmq`/`@prisma/client` 같은 Node 패키지를 직접 deps로 두어 transitive로 `@types/node`가 유입돼 ambient declaration이 자연스럽게 잡혔기 때문. social은 그 경로가 끊겨 있을 뿐.

Node 18+에서는 `fetch`, `URL`, `URLSearchParams`, `Response`, `setTimeout`, `process` 모두 글로벌이고 `@types/node`가 그 타입을 제공함. 따라서 `lib`에 `DOM`을 추가할 필요 **없음** — `@types/node`만 잡히면 28건 전부 해결.

## 변경

### 1. `packages/social/package.json`

```diff
   "devDependencies": {
+    "@types/node": "^20.0.0",
     "@x2/tsconfig": "workspace:*",
     "typescript": "^5.7.0"
   }
```

버전 `^20.0.0`은 워크스페이스의 다른 패키지(workers/analyzer 등)와 정합되는 메이저로 골랐음. 만약 `pnpm why @types/node`로 봤을 때 워크스페이스에 이미 다른 메이저가 자리잡고 있으면 그 메이저로 맞출 것.

### 2. `packages/social/tsconfig.json`

```diff
 {
   "extends": "@x2/tsconfig/library.json",
+  "compilerOptions": {
+    "types": ["node"]
+  },
   "include": ["src"],
   "exclude": ["node_modules", "dist"]
 }
```

`types: ["node"]`로 ambient declaration을 명시. `lib`은 base의 `["ES2022"]` 그대로 유지 (DOM 추가 안 함 — Node 글로벌만으로 충분).

### 왜 base/library.json을 안 건드리는가

`packages/config/typescript/library.json`에 직접 `types: ["node"]`를 박는 옵션도 있지만, 그건 모든 라이브러리 패키지에 영향. 일부 패키지는 의도적으로 Node 글로벌을 안 쓰는 순수 패키지(예: `@x2/types`)일 수 있어서 글로벌 강제는 회귀 위험. **소비자 단위에서 opt-in**이 안전.

## 검증

```bash
pnpm install   # @types/node 추가 후 lockfile 갱신
pnpm --filter @x2/social typecheck 2>&1 | tail -20
# 기대: 0 errors
```

회귀 점검:

```bash
pnpm --filter @x2/api typecheck      # 기대: 1 (pre-existing gdelt-news.adapter.ts)
pnpm --filter @x2/analyzer typecheck # 기대: 0
pnpm --filter @x2/queue typecheck    # 기대: 0
pnpm --filter @x2/web typecheck 2>&1 | grep -c "error TS"  # 기대: 7 (P1-7 스코프, 변동 없음)
pnpm --filter @x2/web build          # 기대: 79/79
```

만약 social typecheck가 28건이 아닌 다른 수가 나오면 (특히 줄어들었지만 0이 아니면) 잔여 카테고리 분류해서 보고. fetch/URL 계열이 0건인데 다른 잔재가 있으면 패키지 자체의 별개 버그일 가능성.

만약 ESM 모듈에서 `process` 사용에 대한 별개 경고(`process` 글로벌 deprecated)가 새로 뜨면 무시하지 말고 보고 — 코드 변경이 추가로 필요할 수 있음.
