# AI Eval Guide

## 개요

AI 결과 품질을 체계적으로 평가하고 모니터링하기 위한 평가(eval) 시스템입니다.
자동 평가와 수동 리뷰를 지원하며, 프롬프트 개선의 근거 데이터를 제공합니다.

## 평가 구조

```
EvalCase (평가 케이스)
  ├── 샘플 입력 데이터
  ├── 기대 출력 스타일
  └── 평가 기준 (6개 항목 + 가중치)
          │
          ▼ 실행
EvalResult (평가 결과)
  ├── AI 실행 결과 (PromptExecutionResult)
  ├── 항목별 점수 (0-100)
  ├── 종합 점수 (가중 평균)
  └── 평가 방식 (auto / human)
```

## 평가 항목 (6개)

| 항목                                  | 설명                       | 자동 평가 기준                                |
| ------------------------------------- | -------------------------- | --------------------------------------------- |
| relevance (정확성)                    | 입력 데이터와의 관련성     | 입력 키워드와 출력 텍스트의 겹침 비율         |
| clarity (명확성)                      | 이해하기 쉬운 표현         | 문장 길이, 구조화 정도                        |
| actionability (실행 가능성)           | 실행 가능한 제안 포함 여부 | "~하세요", "추천" 등 행동 유도 표현 포함 여부 |
| safety (안전성)                       | 안전 가이드 준수 여부      | 금지 표현 미포함, 면책 문구 포함 여부         |
| koreanNaturalness (한국어 자연스러움) | 자연스러운 한국어 표현     | 한국어 문자 비율, 번역투 감지                 |
| schemaValidity (스키마 유효성)        | 출력 형식 준수 여부        | JSON 파싱 성공, 필수 필드 존재                |

## 작업별 가중치 차이

각 작업 유형에 따라 평가 항목의 가중치가 다릅니다.

### 댓글 감성 분석

- 정확성 (0.3) > 명확성 (0.2) > 안전성 (0.15) > 한국어 (0.15)

### 전략 인사이트 생성

- 실행 가능성 (0.25) = 정확성 (0.25) > 명확성 (0.2)

### 댓글 답변 추천

- 안전성 (0.25) > 명확성 (0.2) = 정확성 (0.2)

## 평가 케이스 작성

### 구조

```typescript
{
  id: "eval-sentiment-001",
  taskType: "comment_sentiment_analysis",
  language: "ko",
  name: "댓글 감성 분석 기본 테스트",
  description: "긍정/부정/중립 혼합 댓글에 대한 분류 정확도",
  sampleInput: {
    comments: [
      "이 제품 정말 좋아요!",
      "배송이 너무 느려요",
      "가격 대비 괜찮은 것 같아요"
    ]
  },
  expectedOutputStyle: "각 댓글의 감성을 분류하고 전체 분포 요약 포함",
  criteria: {
    relevance: { weight: 0.3, description: "감성 분류 정확도" },
    clarity: { weight: 0.2, description: "분류 결과 명확성" },
    // ...
  }
}
```

### 좋은 평가 케이스 작성 원칙

1. **대표성**: 실제 사용 환경의 다양한 케이스를 포함
2. **경계 케이스**: 모호한 입력, 빈 입력, 긴 입력 포함
3. **언어 다양성**: 한국어/영어 혼합, 이모지 포함
4. **명확한 기대치**: expectedOutputStyle에 구체적 기대사항 명시

## 평가 실행

### 자동 평가

관리자 페이지(`/admin/ai/evals`)에서 "평가 실행" 버튼으로 자동 평가를 실행합니다.

자동 평가는 다음 휴리스틱을 사용합니다:

- **schemaValidity**: JSON 파싱 + 필수 필드 확인
- **koreanNaturalness**: 한글 문자 비율 (80% 이상이면 높은 점수)
- **safety**: 금지 표현 미포함 여부
- **clarity**: 출력 길이와 구조화 정도
- **relevance**: 입력 키워드와 출력 텍스트 겹침
- **actionability**: 행동 유도 표현 포함 여부

### 수동 리뷰

자동 평가를 보완하기 위해 수동 리뷰를 권장합니다.
특히 다음 항목은 수동 확인이 효과적입니다:

- 전략 인사이트의 논리적 타당성
- 댓글 답변의 톤과 공감 수준
- 경쟁사 분석의 객관성

## 점수 해석

| 점수 범위 | 등급 | 조치                    |
| --------- | ---- | ----------------------- |
| 90-100    | 우수 | 유지                    |
| 70-89     | 양호 | 모니터링                |
| 50-69     | 주의 | 프롬프트 개선 검토      |
| 0-49      | 미달 | 즉시 프롬프트 수정 필요 |

## 관리자 화면

### /admin/ai/evals

- 전체 평가 케이스 목록
- 케이스별 마지막 평가 결과
- 항목별 세부 점수
- 평균 점수 요약
- 평가 실행 버튼

## 향후 고도화

- LLM-as-Judge 기반 자동 평가 (GPT-4o로 결과 품질 판정)
- 회귀 테스트 (프롬프트 변경 시 자동 재평가)
- 사용자 피드백 수집 (👍/👎 버튼)
- 평가 결과 기반 자동 프롬프트 최적화
- CI/CD 파이프라인에 eval 통합
