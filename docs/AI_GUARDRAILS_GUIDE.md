# AI Guardrails Guide

## 개요

AI 출력의 안전성과 품질을 보장하기 위한 다층 가드레일 시스템입니다.
프롬프트 레벨, 출력 검증 레벨, 후처리 레벨의 3단계로 구성됩니다.

## 안전성 레벨

| 레벨     | 적용 대상                             | 설명                        |
| -------- | ------------------------------------- | --------------------------- |
| low      | 분류, 대시보드 설명                   | 기본 검증만 적용            |
| medium   | FAQ, 리포트 요약, 경쟁사 인사이트     | 표준 필터링 적용            |
| high     | 전략 인사이트, 답변 추천, 위험도 평가 | 강화된 검증 + 면책 문구     |
| critical | 의료/법률 관련 (향후 확장)            | 최대 제한, 전문가 권고 필수 |

## 3단계 가드레일

### 1단계: 프롬프트 레벨 가이드

프롬프트 템플릿의 `safetyNote`에 포함되는 지침입니다.

- 확정적 표현 대신 근거 기반 표현 유도
- 의료/법률 등 민감한 주제 제한
- 브랜드 안전 톤 유지 지시
- 개인정보 요청 금지

### 2단계: 출력 검증 (Output Validator)

AI 응답에 대한 자동 검증입니다.

| 검증 항목      | 설명                              |
| -------------- | --------------------------------- |
| JSON 유효성    | 구조화 출력의 파싱 가능 여부      |
| 필수 필드 존재 | summary, confidence 등            |
| 언어 확인      | 한국어 요청 시 한국어 응답 여부   |
| 금지 표현      | "확실합니다", "반드시", "100%" 등 |
| 최소 길이      | 너무 짧은 응답 필터링             |
| 과도한 확신    | 단정적 표현 감지                  |

### 3단계: 후처리 필터 (Safety Guardrails)

출력 정규화 후 적용되는 필터입니다.

- **confidence 상한 제한**: safety level에 따라 최대 표시 confidence 제한
  - high safety: 최대 0.85
  - medium safety: 최대 0.95
  - low safety: 1.0 (제한 없음)
- **면책 문구 자동 추가**: high/critical safety 작업에 면책 문구 부착
- **금지 표현 치환**: 과도한 확신 표현을 완화된 표현으로 대체
- **민감 콘텐츠 플래그**: 의료, 법률, 금융 관련 내용 감지 시 riskFlag 추가

## 금지 표현 목록

| 금지 표현           | 대체 표현              |
| ------------------- | ---------------------- |
| 확실합니다          | ~로 판단됩니다         |
| 반드시 ~해야 합니다 | ~하는 것을 권장합니다  |
| 100% 보장           | 높은 가능성이 있습니다 |
| 절대적으로          | 대체로                 |
| 틀림없이            | ~로 보입니다           |

## 작업별 안전 정책

### 댓글 답변 추천 (reply_suggestion_generation)

- Safety: **high**
- 브랜드 톤 유지 필수
- 논쟁적 발언 금지
- 개인정보 요청 금지
- 공감적, 전문적 톤 유도

### 전략 인사이트 (strategy_insight_generation)

- Safety: **high**
- 데이터 근거 필수
- 확정적 예측 금지
- "~할 수 있습니다", "~로 보입니다" 표현 유도
- 면책 문구 포함

### 경쟁사 인사이트 (competitor_insight_generation)

- Safety: **medium**
- 근거 없는 비방 금지
- 확인되지 않은 정보 포함 금지
- 객관적 데이터 비교 중심

### 댓글 위험도 평가 (comment_risk_assessment)

- Safety: **high**
- 혐오 표현, 인신공격 정확히 감지
- 위양성(false positive) 최소화
- 위험 등급 근거 명시

## 검증 실패 시 처리

```
검증 실패 → retry (simpler prompt) → fallback provider → mock 응답
```

1. 첫 번째 시도 실패: 더 단순한 프롬프트로 재시도
2. 두 번째 시도 실패: fallback provider로 전환
3. 모든 시도 실패: mock 응답 또는 에러 반환

## 면책 문구 (Disclaimer)

high/critical safety 작업의 출력에 자동 추가됩니다:

> "이 분석은 AI가 생성한 참고 자료이며, 전문가의 판단을 대체하지 않습니다."

## 모니터링

`/admin/ai/logs`에서 검증 실패 로그를 확인할 수 있습니다.
`/admin/ai/evals`에서 안전성 점수를 포함한 품질 평가를 실행할 수 있습니다.
